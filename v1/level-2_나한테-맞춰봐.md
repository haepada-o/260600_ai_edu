## 0. 이 문서는 누구를 위한 것인가

이 문서는 **세 가지 조건**에 해당하는 분을 위해 만들었습니다.

| 조건 | 설명 |
|------|------|
| ✅ 레벨 1을 마친 사람 | AI와 대화하는 기본기를 익혔다 |
| ✅ "매번 같은 걸 설명하는 게 귀찮다"는 사람 | 새 채팅마다 "나는 마케터이고…"를 반복하고 있다 |
| ✅ 도구를 설치해서 쓸 의향이 있는 사람 | 웹 채팅만으로는 한계를 느꼈다 |

> [!NOTE]
> 레벨 1에서는 **"어떻게 말하느냐"**가 핵심이었습니다.  
> 레벨 2에서는 **"어떻게 기억시키느냐"**가 핵심입니다.

**이 문서를 마치면 할 수 있는 것:**
- 🔧 AI에게 나만의 규칙을 가르치고, 매번 반복 설명 없이 일할 수 있다
- 🤖 자주 하는 업무를 자동화하는 커스텀 명령어를 만들 수 있다
- 💻 (선택) 내 컴퓨터에서 직접 AI를 돌릴 수 있다
- 🔌 (선택) AI에 외부 도구를 연결하는 MCP 개념을 이해한다

## 1. 규칙 파일: AI에게 나를 가르치는 법

### 문제: AI는 나를 모른다

새 채팅을 열 때마다, AI는 당신을 처음 만납니다.

```
❌ 매번 하는 말:
"나는 부동산 VC 대표이고, 펀드 운용을 하고 있어.
 금액은 억원 단위로, 면적은 평으로 표기해줘.
 수치에는 반드시 근거를 붙여줘..."
```

10번, 20번 반복하면 **시간 낭비**이고, 빼먹으면 AI가 엉뚱한 결과를 냅니다.

### 해결: 규칙 파일을 만들면 AI가 매번 읽는다

**규칙 파일**이란, AI가 대화를 시작할 때 **자동으로 읽는 설정 문서**입니다.

한 번 만들어두면:
- ✅ 새 채팅을 열어도 AI가 나를 안다
- ✅ 매번 설명을 반복할 필요가 없다
- ✅ 답변의 일관성이 올라간다

> [!IMPORTANT]
> 핵심 개념: 규칙 파일은 AI의 **"신입사원 매뉴얼"**입니다.  
> 새로 입사한 직원에게 "우리 회사는 이렇게 일해"라고 한 번 알려주면,  
> 매번 가르칠 필요가 없는 것과 같습니다.

### 실제 사례 — 547줄짜리 CLAUDE.md

실제로 한 사용자는 **547줄짜리 규칙 파일**을 만들어 사용하고 있습니다. 이 파일에는 다음이 포함되어 있습니다:

| 섹션 | 내용 | 효과 |
|------|------|------|
| 업무 분류 시스템 | 기획 / 관리 / 리서치 구분 | AI가 요청의 성격을 자동 판단 |
| 에이전트 라우팅 규칙 | 어떤 요청을 어떤 에이전트에게 보내는지 | 역할 분배 자동화 |
| 데이터 접근 원칙 | 어떤 파일을 참조해도 되고, 안 되는지 | 보안 + 정확성 확보 |
| 프로젝트 대시보드 | 진행 중인 프로젝트 목록과 상태 | 맥락 파악 시간 제로 |

> **"AI를 '그냥 물어보는 도구'로 쓰는 것과, '나의 시스템을 관리하는 에이전트'로 쓰는 것의 차이입니다."**

처음부터 547줄을 쓸 필요는 없습니다. **10줄짜리 규칙부터 시작**해서 점점 키워가면 됩니다.

### 각 서비스에서 규칙 설정하는 법

| 서비스 | 규칙 이름 | 설정 위치 | 글자 제한 | 특징 |
|--------|----------|----------|----------|------|
| **ChatGPT** | 사용자 지정 지침 | 설정 → 개인화 → 사용자 지정 지침 | ~1,500자 | 모든 채팅에 자동 적용 |

> - 가장 유연한 것: **Gemini Gems** (용도별로 여러 개 만들기 좋음)

### 규칙 파일 템플릿 (초보자용)

아래 템플릿을 복사해서 자신에 맞게 수정하세요. **괄호 안의 내용만 바꾸면 됩니다.**

```markdown
# 나의 AI 규칙

## 나는 누구인가
- 이름: [이름]
- 직업: [직업]
- 주로 하는 일: [업무 설명]

## AI에게 바라는 것
- 존댓말/반말: [선택]
- 답변 길이: [짧게/보통/길게]
- 톤: [전문적/친근/간결]

## 내 업무 규칙
- [규칙 1: 예) 수치를 쓸 때는 반드시 출처를 밝혀라]
- [규칙 2: 예) 제안서에는 과장된 표현을 쓰지 마라]
- [규칙 3: 예) 이메일은 3문장 이내로]

## 자주 쓰는 용어
- [용어 1] = [의미]
- [용어 2] = [의미]
```

> [!WARNING]
> **흔한 실수: 너무 많이 쓰려고 하는 것**  
> 처음에는 5~10줄이면 충분합니다. 사용하면서 "아, 이것도 알려줘야겠다"  
> 싶은 것을 하나씩 추가하세요. 규칙 파일은 **살아있는 문서**입니다.

### 실전 예시 — 부동산 VC 대표님용 규칙 파일

실제 업무에서 사용 중인 규칙 파일의 예시입니다:

```markdown
# 나의 AI 규칙

## 나는 누구인가
- 직업: 부동산/해양 분야 VC 대표
- 주요 업무: 펀드 운용, 투자 심사, 제안서 작성, LP 보고

## 업무 규칙
- 수치는 반드시 근거를 함께 제시
- 면적 단위: 평 기준 (㎡ 병기)
- 금액: 억원 단위, 천만원 이하 절사
- Co-GP 정보는 프로젝트마다 다르므로 하드코딩 금지

## 자주 쓰는 용어
- SPC = 특수목적법인
- LP = 출자자 (Limited Partner)
- GP = 운용사 (General Partner)
- 밸류업 = 피투자기업 성장 지원
- AUM = 운용자산총액 (Assets Under Management)
- 드라이파우더 = 투자 가용 자금
- 클로징 = 펀드/거래 최종 완료
```

**이 규칙 파일이 만드는 차이:**

| | 규칙 없이 | 규칙 적용 후 |
|---|----------|-------------|
| 금액 표기 | "약 3,000만원" | "30억원 (천만원 이하 절사)" |
| 면적 표기 | "1,000㎡" | "302평 (1,000㎡)" |
| 수치 제시 | "부동산 시장이 성장하고 있습니다" | "국토부 2025 통계 기준 수도권 오피스 공실률 8.2%" |
| 용어 사용 | "투자받는 회사" | "SPC (특수목적법인)" |

### Before & After: 규칙 파일의 효과

**📋 같은 질문, 다른 결과**

**질문**: "이번 펀드 운용 보고서 초안 만들어줘"

#### ❌ 규칙 파일 없이:
```
펀드 운용 보고서

1. 개요
본 펀드는 투자자들에게 안정적인 수익을 제공하기 위해 운용되고 있습니다.
투자 대상은 다양한 자산으로 구성되어 있으며...

(→ 너무 일반적, 업계 용어 없음, 양식도 모름)
```

#### ✅ 규칙 파일 적용 후:
```
B-Square 부동산펀드 제3호 운용 보고서

■ 펀드 개요
- AUM: 150억원
- 운용사(GP): 비스퀘어자산운용
- LP 구성: 기관 3사

■ 운용 현황
- 투자 집행: 95억원 (집행률 63.3%)
- 드라이파우더: 55억원
- 클로징 예정: 2건 (합산 42억원, 7월 예정)

■ 포트폴리오 현황
| 자산명 | 면적 | 취득가 | 현재 감정가 |
|--------|------|--------|-----------|
| ○○오피스 | 450평 (1,488㎡) | 38억원 | 42억원 |

(→ 업계 양식, 정확한 용어, 규칙에 맞는 단위)
```

## 2. 커스텀 명령어: 자주 하는 일을 한 번에

규칙 파일이 **"나는 누구인가"**를 알려주는 것이라면,  
커스텀 명령어는 **"이 일을 이렇게 해라"**를 알려주는 것입니다.

| 개념 | 비유 | 역할 |
|------|------|------|
| 규칙 파일 | 신입사원 매뉴얼 | "우리 회사는 이렇게 일해" |
| 커스텀 명령어 | 업무 매뉴얼 | "이 보고서는 이 순서로 써" |

### Gemini Gems 예시: "투자 분석가" Gem 만들기

**Gemini Gems**는 특정 목적에 맞는 **맞춤 AI 챗봇**을 만드는 기능입니다.

#### Step 1: Gem Manager 열기
1. gemini.google.com 접속
2. 왼쪽 메뉴에서 **"Gem Manager"** 클릭
3. **"새 Gem 만들기"** 클릭

#### Step 2: 기본 정보 설정
- **이름**: 투자 분석가
- **설명**: 부동산/인프라 투자 기회를 분석하고 투자 심사 보고서를 작성합니다

#### Step 3: 규칙 입력
아래 내용을 **지침(Instructions)** 영역에 붙여넣기:

```
너는 부동산/인프라 분야 투자 분석가다.

[역할]
- 투자 기회를 분석하고 투자 심사 보고서를 작성한다
- 리스크 요인을 빠짐없이 도출한다
- 시장 데이터에 근거한 의견을 제시한다

[규칙]
- 금액: 억원 단위 (천만원 이하 절사)
- 면적: 평 기준 (㎡ 병기)  
- 수익률: 연 환산 IRR 기준
- 모든 수치에 출처 또는 산출 근거 필수
- 리스크는 최소 3개 이상 도출

[출력 양식]
1. 투자 개요 (한 문단)
2. 시장 분석 (데이터 기반)
3. 수익성 분석 (IRR, 배수)
4. 리스크 분석 (3개 이상)
5. 종합 의견 (투자/보류/기각)
```

#### Step 4: 테스트하기
Gem을 저장한 후, 다음과 같이 테스트:

```
"서울 강남구 역삼동 오피스 빌딩, 
 매입가 200억, 연 임대수익 16억, 
 5년 보유 후 매각 예상가 240억.
 분석해줘."
```

✅ **제대로 작동하면**: 양식에 맞춰, 억원 단위로, IRR과 리스크를 포함한 보고서가 나옵니다.

### ChatGPT GPTs 예시: "제안서 검토기" 만들기

**GPTs**는 ChatGPT에서 만드는 맞춤 AI입니다. 대화형 설정으로 쉽게 만들 수 있습니다.

#### Step 1: GPT 만들기 시작
1. chatgpt.com → 왼쪽 메뉴 **"GPT 탐색"**
2. 오른쪽 상단 **"+ 만들기"** 클릭

#### Step 2: "만들기" 탭에서 대화로 설정
ChatGPT가 "어떤 GPT를 만들고 싶으세요?"라고 물어봅니다.

```
제안서를 검토하는 GPT를 만들고 싶어.
- 제안서를 올리면 체크리스트 기반으로 검토해줘
- 빠진 항목, 논리적 비약, 수치 오류를 찾아줘
- 결과를 표로 정리해줘
```

#### Step 3: "구성" 탭에서 세부 조정
- **이름**: 제안서 검토기
- **설명**: 제안서의 완성도를 체크리스트 기반으로 검토합니다
- **지침**에 추가:

```
[체크리스트]
□ 사업 목적이 명확한가
□ 목표 수익률의 산출 근거가 있는가
□ 리스크 요인이 3개 이상 기술되었는가
□ 경쟁 우위가 구체적인가 (vs 추상적 표현)
□ 투자 금액과 회수 계획이 일치하는가
□ 오탈자 및 수치 불일치가 없는가

[검토 결과 양식]
| 항목 | 상태 | 코멘트 |
|------|------|--------|
| ... | ✅/⚠️/❌ | ... |
```

#### Step 4: 지식 파일 업로드 (선택)
- 과거 성공 제안서 예시 (PDF)
- 사내 제안서 양식 (Word)
- 평가 기준표

#### Step 5: 저장 및 테스트
저장 후, 실제 제안서를 업로드하여 테스트합니다.

### Claude Projects 예시: "펀드 관리" 프로젝트 만들기

**Claude Projects**는 가장 강력한 규칙 설정 방식입니다. 파일 업로드 + 긴 지침이 가능합니다.

#### Step 1: 새 프로젝트 만들기
1. claude.ai 접속
2. 왼쪽 사이드바에서 **"Projects"** 클릭
3. **"Create a new project"** 클릭
4. 이름: **"펀드 관리"**

#### Step 2: 지식 파일 업로드
프로젝트 설정에서 **"Add knowledge"** 클릭 후 업로드:

| 업로드할 파일 | 용도 |
|-------------|------|
| 펀드 구조도.pdf | AI가 펀드 구조를 이해 |
| LP 목록.xlsx | 투자자 정보 참조 |
| 용어집.md | 업계 전문용어 정의 |
| 과거 보고서 예시.pdf | 양식과 톤 학습 |

> [!TIP]
> Claude는 **200K 토큰** (약 15만 단어, 단행본 2~3권 분량)까지  
> 지식으로 넣을 수 있습니다. 충분히 많은 맥락을 제공하세요.

#### Step 3: 커스텀 지침 설정
**"Set custom instructions"** 클릭 후 입력:

```
이 프로젝트는 B-Square자산운용의 펀드 관리를 위한 것입니다.

[기본 규칙]
- 모든 금액: 억원 단위 (천만원 이하 절사)
- 면적: 평 기준 (㎡ 병기)
- 수익률: 연 환산 IRR
- LP 관련 정보: 업로드된 LP 목록 참조
- 보고서 양식: 업로드된 과거 보고서 예시 따를 것

[자주 요청하는 작업]
1. LP 분기 보고서 초안
2. 투자 심사 보고서
3. 펀드 현황 요약
4. 공고 분석 및 대응 전략
```

#### Step 4: 프로젝트 채팅 시작
프로젝트 안에서 새 채팅을 시작하면, **업로드한 파일과 지침이 자동으로 적용**됩니다.

```
"이번 분기 LP 보고서 초안 만들어줘. 
 ○○펀드 기준, 3분기 실적 반영해서."
```

→ AI가 LP 목록, 과거 보고서 양식, 용어집을 모두 참조하여 초안을 생성합니다.

## 3. 로컬 설치: 내 컴퓨터에서 AI 쓰기

> [!NOTE]
> 이 섹션은 **선택사항**입니다. 웹에서 충분히 잘 쓰고 있다면 건너뛰어도 됩니다.  
> 하지만 **데이터 보안**이 중요하거나, **더 깊은 활용**을 원한다면 읽어보세요.

### 왜 로컬인가?

> **"로컬에서 설치해서 쓰면 내 컴퓨터에 있고 쟤한테는 없어요. 데이터가."**

| | 웹 서비스 (ChatGPT 등) | 로컬 설치 |
|---|---------------------|----------|
| 데이터 위치 | 서버 (미국 등) | 내 컴퓨터 |
| 인터넷 필요 | ✅ 필수 | ❌ 오프라인 가능 (일부) |
| 파일 접근 | 업로드 필요 | 직접 접근 |
| 비용 | 월 구독료 | 무료~월 구독료 |
| 보안 | 서비스 정책에 의존 | 완전 통제 |

### 옵션 비교

| 도구 | 가격 | 난이도 | 장점 | 단점 |
|------|------|--------|------|------|
| **Antigravity** | $20/월~ | ★★☆ | Google 생태계 연동, 서브에이전트 | 새 서비스 |
| **Claude Code** | $20/월~ | ★★★ | 파일 편집 최강, 코드 실행 | CLI 기반 |
| **Cursor** | $20/월~ | ★★☆ | 다양한 AI 모델 선택, 시각적 UI | 코딩 중심 |
| **Ollama** | **무료** | ★★★ | 완전 로컬, 데이터 유출 없음 | GPU 필요 |

> [!TIP]
> **추천 시작점:**
> - 코딩을 전혀 모른다 → **Antigravity** (가장 친절한 UI)
> - 보안이 최우선이다 → **Ollama** (완전 로컬, 무료)
> - 코딩을 좀 한다 → **Claude Code** 또는 **Cursor**

---

### Antigravity — 채팅처럼 시작, 시스템까지 확장

**Antigravity**는 Google이 만든 AI 도구입니다. 코딩만 하는 게 아닙니다.

**왜 Antigravity인가?**
- **채팅 모드 + 코딩 모드** 둘 다 있다
- **서브에이전트** — AI가 다른 AI에게 일을 시킬 수 있다 (동시 병렬 작업)
- **폴더 기반** — 내 컴퓨터의 파일을 AI가 직접 읽고 수정
- **스킬/규칙** — 반복 작업을 자동화하는 시스템 내장

**설치**: gemini.google.com/app → Antigravity 다운로드 → Pro 이상 결제 시 Claude, Gemini, GPT 모두 사용 가능

**실제 사용 예시**:
```
[채팅 모드] — 일반 대화처럼
"이번 주 해양수산 뉴스 5개 요약해줘"

[IDE 모드] — 폴더 열고 작업
"이 폴더에 있는 제안서 초안을 배점 기준에 맞춰서 검토해줘"
```

**핵심 장점**: 여기서 구축한 시스템(규칙, 스킬)을 나중에 Claude Code나 Cursor로 **거의 그대로 가져갈 수 있다**.

> "클로드에서 밴 먹어서 안티그래비티로 옮겼는데, 세팅해 놓은 값을 그대로 가져와서 똑같이 세팅 가능해요."

---

### Claude Code — 정밀한 파일 작업의 끝판왕

**Claude Code**는 터미널(명령줄)에서 돌아가는 AI입니다. 화면은 심플하지만 기능은 가장 강력합니다.

**왜 Claude Code인가?**
- **다중 파일 동시 편집** — 제안서 10개를 한 번에 수정
- **터미널 실행** — 엑셀 변환, PDF 생성 등 직접 실행
- **CLAUDE.md** — 프로젝트별 규칙 파일이 자동 로드

**설치**:
```bash
npm install -g @anthropic-ai/claude-code
cd ~/내프로젝트폴더
claude
```

**실제 사용 예시**:
```
# 터미널 1 — 제안서 작업
cd ~/B-Square && claude
> "@skills/펀드출자제안서.md 결성배경 섹션 작성해줘"

# 터미널 2 — 동시에 리서치
cd ~/B-Square && claude
> "@agents/researcher.md 모태펀드 최신 정책 트렌드 수집해줘"
```

---

### Cursor — 시각적 편집 + AI

**Cursor**는 코드 에디터(VS Code 기반)에 AI를 탑재한 것입니다. 파일을 눈으로 보면서 작업합니다.

**왜 Cursor인가?**
- **시각적 UI** — 파일 탐색기, 미리보기 등 눈으로 보면서 작업
- **다양한 모델** — Claude, GPT, Gemini 등 한 곳에서
- **`@` 참조** — `@agents/researcher.md` 처럼 규칙 파일을 직접 참조

**설치**: cursor.com → 다운로드 → Pro ($20/월) 결제

---

### 하이브리드 워크플로우 — 하나만 쓰지 않는다

```
[큰 그림 잡기]          → Antigravity (서브에이전트로 동시 리서치)
          ↓
[정밀 파일 작업]        → Claude Code (다중 파일 편집, 터미널 실행)
          ↓
[시각적 확인·미세 수정]  → Cursor (UI로 파일 탐색, @ 참조)
          ↓
[검증·브라우저 테스트]   → Antigravity (브라우저 자동화)
```

**실제 제안서 작업 예시**:

| 단계 | 도구 | 하는 일 |
|------|------|---------|
| 1. 공고 분석 | Antigravity | 서브에이전트 2개로 공고문 + 경쟁사 동시 분석 |
| 2. 초안 작성 | Claude Code | 터미널 2개로 제안서 + 리서치 병렬 |
| 3. 수치 검증 | Cursor | 엑셀 열어서 수치 눈으로 확인하며 수정 |
| 4. 최종 검토 | Claude Code | CLAUDE.md 규칙 기반 자동 검수 |

**토큰 분산 전략**:
- Antigravity rate limit 도달 시 → Claude Code로 전환
- Claude Code context 한계 시 → Antigravity에 요약 전달
- 대규모 문서 분석 → Antigravity (1M context window)
- 정밀 편집 → Claude Code

**핵심**: 도구가 바뀌어도 **CLAUDE.md, agents/, skills/ 같은 시스템 파일은 거의 그대로 이식 가능**.
Antigravity에서 Cursor로, Claude Code에서 Antigravity로 — 시스템은 유지됩니다.

---

### 유튜브로 시스템을 지속 개선하는 방법

로컬 도구를 설치했으면, **끊임없이 개선**해야 합니다.

```
① 유튜브에서 설정 강의 찾기
   ("claude code 워크플로우", "cursor 설정", "antigravity skills" 등)
         ↓
② NotebookLM에 유튜브 링크 넣기 → 자막 텍스트 자동 추출
         ↓
③ AI에게 "이 강의 내용과 내 현재 CLAUDE.md를 비교해줘" 요청
         ↓
④ 차이점만 내 시스템에 반영
```

**실제 사례**: 유튜브에서 "스킬 설정법" 강의 텍스트 추출 → 내 에이전트 설정과 비교 → "데이터 신선도 규칙"이 빠져있음을 발견 → 내 규칙에 추가

> **핵심**: 남의 시스템을 통째로 복사하지 않는다.
> 추출 → 비교 → 차이점만 흡수. 이걸 반복하면 시스템이 계속 진화한다.

### Ollama 설치 가이드 (완전 무료, 로컬)

**Ollama**는 내 컴퓨터에서 AI 모델을 직접 돌리는 도구입니다. 무료이고, 인터넷 없이도 작동합니다.

물론입니다. 투자 심사 보고서의 기본 양식을 제안드리겠습니다...
```

## 4. MCP: 외부 도구 연결

> [!NOTE]
> MCP는 **Model Context Protocol**의 약자입니다.  
> 이름은 어렵지만, 개념은 간단합니다.

### MCP란 무엇인가

**MCP = AI가 다른 프로그램을 직접 사용할 수 있게 하는 연결 장치**

#### 비유: AI에게 팔을 달아주는 것

```
🧠 기본 AI = 말만 할 수 있는 뇌
   "구글 드라이브에서 파일 찾아줘" → "저는 드라이브에 접근할 수 없습니다"

🦾 MCP 연결 후 = 뇌 + 팔
   "구글 드라이브에서 파일 찾아줘" → (직접 접근) → "이 파일 말씀하시는 거죠?"
```

| | MCP 없이 | MCP 연결 후 |
|---|---------|------------|
| Google Drive | 파일을 복사해서 붙여넣기 | AI가 직접 읽기 |
| 엑셀/스프레드시트 | 데이터를 복사해서 입력 | AI가 직접 수정 |
| 캘린더 | 일정을 말로 알려주기 | AI가 직접 확인 |
| 한글 문서 | .hwp 파일 열 수 없음 | AI가 읽기/쓰기 가능 |
| Notion | 페이지 내용 복사 | AI가 직접 관리 |

---

### 실제 사용 예시

한 사용자가 실제로 연결해서 쓰고 있는 MCP 구성:

```
실제 MCP 연결 구성:

📁 bsquare-drive MCP  → Google Drive 파일 읽기
   "드라이브에서 '○○펀드 제안서' 찾아서 요약해줘"

📄 hwpx MCP          → 한글 문서 읽기/쓰기  
   "이 한글 파일 읽어서 표 데이터 뽑아줘"

📝 notion MCP        → Notion 페이지 관리
   "이번 주 회의록을 Notion에 정리해줘"

📊 google-sheets MCP → 스프레드시트 읽기/쓰기
   "포트폴리오 시트에서 수익률 계산해줘"
```

> **"MCP 검색해보면 되게 이상하고 신기한 것들 되게 많이 나오거든요."**

---

### 사용 가능한 MCP 예시

현재 다양한 MCP가 공개되어 있습니다:

| MCP | 기능 | 활용 예 |
|-----|------|--------|
| Google Drive | 드라이브 파일 읽기/쓰기 | 보고서 자동 작성 |
| Google Sheets | 스프레드시트 조작 | 데이터 분석, 차트 생성 |
| Google Calendar | 일정 관리 | 회의 일정 자동 정리 |
| Notion | 노션 페이지 관리 | 프로젝트 관리, 위키 |
| Slack | 메시지 관리 | 채널 요약, 알림 |
| GitHub | 코드 저장소 | 이슈 관리, PR 작성 |
| HWPX | 한글 문서 | 한글 파일 읽기/쓰기 |
| Brave Search | 웹 검색 | 실시간 정보 검색 |
| File System | 로컬 파일 | 컴퓨터 파일 직접 접근 |

> [!WARNING]
> **MCP 연결은 현재 기술적 설정이 필요합니다.**  
> JSON 설정 파일을 편집해야 하는 경우가 많아, 비개발자에겐 어려울 수 있습니다.  
> 하지만 **개념을 이해하는 것**만으로도 레벨 3로 나아가는 데 충분합니다.

---

### MCP의 미래

지금은 설정이 어렵지만, 점점 쉬워지고 있습니다:

```
