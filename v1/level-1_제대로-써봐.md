### 맥락을 주고, 거절하고, 저장하는 법

## 0. 이 문서는 누구를 위한 것인가

### 이런 분이 읽어야 합니다

- 레벨 0을 마친 사람 — 모델 선택, 기본 입력은 할 줄 안다
- AI가 답을 주긴 하는데, **"이게 내가 원하는 게 아닌데…"** 경험이 있는 사람
- 결과는 나오는데 **뭔가 얕다**, **뻔하다**, **교과서 같다**고 느끼는 사람

### 이 문서의 핵심 관점

> **"AI에게 정답을 묻는 것이 아니라, 사고 과정을 함께 설계한다."**

레벨 0에서는 AI에게 **요청하는 법**을 배웠습니다.
레벨 1에서는 AI와 **대화하는 법**을 배웁니다.

차이가 뭐냐고요?

- 요청: "이거 해줘" → 결과를 받는다
- 대화: "이런 상황인데" → 같이 생각한다 → 아닌 건 거절한다 → 더 좋은 결과가 나온다

---

## 1. 맥락 먼저, 요청 나중에

### 대부분의 사람은 이렇게 합니다

> ❌ "시나리오 써줘"

그리고 나온 결과를 보고 실망합니다.
*"이게 뭐야, 너무 뻔한데."*

당연합니다.
AI는 여러분이 **어떤 시나리오를 원하는지** 전혀 모릅니다.
그래서 **세상에서 가장 무난한** 시나리오를 줍니다.

---

### 좋은 예: 실제 사례 — 메카노림픽 프로젝트의 시작

이것은 실제로 AI에게 입력된 첫 문장입니다:

> "인간들의 올림픽. 사람들은 달린다. 들어올린다. 쏜다. 부대낀다.
> 도핑으로 얼룩진 올림픽은 이미 인간의 한계를 의심하고 있다."

이게 프롬프트인가요? **아닙니다.**
이건 **생각**입니다. 일기에 가깝습니다.

그런데 AI는 이 생각에 반응했습니다.
명령이 아니라 **맥락**에 반응한 겁니다.

> **"이건 프롬프트라기보다 일기에 가깝죠.**
> **AI에게 '시나리오 써줘'라고 하지 않았습니다."**

AI에게 **명령**을 하면 → 무난한 결과가 나옵니다.
AI에게 **생각을 공유**하면 → 그 생각을 기반으로 같이 만들어갑니다.

---

### 연습 템플릿

AI에게 요청할 때, 이 네 가지 칸을 채워보세요:

```
[상황] 나는 지금 ~하고 있다
[생각] 이런 느낌/질문이 있다
[방향] 이런 쪽으로 가고 싶다
[제약] 이건 하지 마
```

> [!TIP]
> **네 칸을 모두 채울 필요는 없습니다.**
> 하지만 최소한 **[상황]과 [방향]**은 넣어주세요.
> 이 두 가지만 있어도 결과가 완전히 달라집니다.

---

### 실전 예시 3개

#### 예시 1: 창작

```
[상황] 단편 소설을 쓰려고 한다
[생각] 사람이 기계와 경쟁하는 올림픽이라는 이미지가 있다
[방향] 디스토피아가 아니라 의문을 던지는 톤
[제약] 결론을 내리지 마. 답을 주지 마.
```

이렇게 입력하면, AI는 "감동적인 로봇 올림픽 이야기"를 쓰지 않습니다.
대신 **"왜 인간은 아직 뛰는가?"** 같은 질문을 던지기 시작합니다.

---

#### 예시 2: 사업

```
[상황] 해양수산 분야 창업지원 프로그램 제안서를 써야 한다
[생각] 우리 회사 강점은 실제 투자 실적이 있다는 것
[방향] 심사위원이 "이 팀이라면 된다"고 느끼게
[제약] 과장하지 마. 수치는 내가 준 것만 써.
```

이렇게 하면 AI가 멋대로 숫자를 만들어내지 않습니다.
**여러분이 준 데이터 안에서** 가장 설득력 있는 구조를 만듭니다.

---

#### 예시 3: 일상

```
[상황] 직원 면담을 해야 하는데 분위기가 안 좋다
[생각] 상대방이 어제 불만을 표출했다
[방향] 감정을 인정하되 업무 방향은 잡아야 한다
[제약] 위로만 하는 말 금지. 구체적 액션이 있어야 한다.
```

이 제약 조건이 없으면 AI는 이런 답을 줍니다:
*"상대방의 감정을 충분히 공감해주시고, 따뜻한 분위기에서 대화하세요..."*

제약을 넣으면:
*"1단계: '어제 이야기 잘 들었다. 그 부분은 나도 문제라고 생각한다.'로 시작. 2단계: '다만 이번 주 안에 X를 처리해야 하는 상황이다. 어떻게 하면 가능할까?'로 연결. 3단계: 구체적 일정과 역할을 합의."*

**제약이 결과를 바꿉니다.**

## 2. AI의 뻔한 답을 거절하라

### 왜 거절해야 하는가

AI의 기본값은 **'무난하고 긍정적'**입니다.
이건 AI가 나빠서가 아니라, 그렇게 학습되었기 때문입니다.

AI는 기본적으로:
- 긍정적으로 답하려고 합니다
- 결론을 내리려고 합니다
- 예쁘게 마무리하려고 합니다

하지만 여러분이 원하는 건 보통 **'날카롭고 구체적'**인 것입니다.

**그래서 거절해야 합니다.**

---

### 실제 사례: 메카노림픽

메카노림픽 프로젝트에서, AI가 첫 번째로 내놓은 결과물에는 이런 내용이 있었습니다:

*"기술과 인간의 조화를 통해 새로운 올림픽의 가능성을 엿보다..."*

이 문장은 **어떤 글에 넣어도 되는** 말입니다.
즉, **아무 의미도 없는** 말입니다.

거절했습니다.

> **"의문만 던지도록 해줘. 아름다운 그림은 필요 없어."**

AI가 결론 대신 **질문을 던지기** 시작했습니다:
- *"기계가 인간보다 빠르다면, '빠름'은 여전히 위대한가?"*
- *"도핑을 금지하면서 의족은 허용하는 기준은 무엇인가?"*

세계관이 깊어졌습니다.
**거절 한 번이 품질을 바꿨습니다.**

---

### 거절 문장 10선

이것들을 복사해서 쓰세요. 외울 필요 없습니다.
AI가 뻔한 답을 줄 때마다 하나씩 꺼내 쓰면 됩니다.

| # | 거절 문장 | 언제 쓰나 |
|---|----------|----------|
| 1 | **"이거 너무 뻔하다. 다시."** | 결과가 교과서 같을 때 |
| 2 | **"결론 내리지 마. 질문만 던져."** | AI가 억지로 마무리할 때 |
| 3 | **"칭찬하지 마. 문제점만 말해."** | 내 아이디어를 평가받고 싶을 때 |
| 4 | **"교과서적인 답 말고, 실제로 이 상황에서 뭐가 위험한지."** | 현실적인 리스크를 알고 싶을 때 |
| 5 | **"이 방향 아니야. 정반대로 가봐."** | 완전히 다른 관점이 필요할 때 |
| 6 | **"너무 길어. 핵심 3줄로."** | 장문 답변에 지칠 때 |
| 7 | **"이건 내가 이미 아는 거야. 내가 모르는 걸 알려줘."** | 새로운 인사이트가 필요할 때 |
| 8 | **"아름다운 그림 필요 없어. 불편한 진실만."** | 솔직한 피드백이 필요할 때 |
| 9 | **"이 단어 쓰지 마: 혁신, 시너지, 패러다임."** | 버즈워드를 걷어낼 때 |
| 10 | **"네가 방금 한 말의 반대 주장을 만들어봐."** | 양쪽 관점을 다 보고 싶을 때 |

> [!IMPORTANT]
> **거절은 무례한 게 아닙니다.**
> AI에게 "이거 별로야"라고 하는 건 AI의 감정을 상하게 하지 않습니다.
> 거절은 **여러분의 기준을 AI에게 가르치는 것**입니다.

---

### Before / After

#### Before: 거절 안 함

입력: "올림픽 관련 소설 써줘"

> 결과: 젊은 선수가 역경을 이겨내고 금메달을 따는 감동 스토리.
> "그는 땀과 눈물 끝에 마침내 꿈을 이루었다. 관중석의 환호가..."

**어디서 많이 본 이야기입니다.**

---

#### After: 거절 3번

1차 거절: "결론 내리지 마."
→ AI가 결말을 열어둔 이야기를 씀

2차 거절: "아름다운 그림 없어."
→ 감동적인 묘사가 사라지고, 건조한 기록 문체가 됨

3차 거절: "의문만 던져."
→ AI가 이야기 대신 질문을 배치하기 시작

> 최종 결과: 인간 선수와 기계 선수가 같은 트랙에 서 있다.
> 기계가 이겼다. 관중은 박수를 친다.
> "누구에게 박수를 치는 건가? 기계에게? 기계를 만든 엔지니어에게?
> 아니면 졌는데도 뛴, 인간에게?"

**세 번 거절했더니, 세계관이 깊은 작품이 되었습니다.**

## 3. 모델별로 역할을 나눠라

### 비유: 직원마다 잘하는 게 다르다

회사에서 모든 일을 한 사람에게 시키지 않습니다.
디자인은 디자이너에게, 회계는 회계사에게.

**AI도 마찬가지입니다.**

ChatGPT, Claude, Gemini, Perplexity — 각각 잘하는 게 다릅니다.
하나만 쓰면 **70점**입니다.
적재적소에 쓰면 **95점**입니다.

---

### AI별 강점 비교

| 할 일 | 추천 AI | 왜? |
|-------|---------|-----|
| 녹음 → 텍스트 | **NotebookLM** (Gemini) | 무료, 마인드맵까지 자동 생성 |
| 이미지 만들기 | **ChatGPT** | 이미지 생성이 가장 자연스러움 |
| 정밀한 문서 | **Claude** | 구조화 능력, 디테일 최고 |
| 큰 그림 잡기 | **ChatGPT** | 발상 넓히기, 브레인스토밍에 좋음 |
| 검색 + 팩트체크 | **Perplexity** | 출처를 같이 보여줌 |
| 코드 / 자동화 | **Claude** | 정확도 높음 |
| 법률 / 세무 초벌 | **Claude** | 단, 반드시 전문가 검증 필요 |

> [!WARNING]
> **법률/세무 관련은 AI 결과를 절대 그대로 쓰지 마세요.**
> AI가 주는 건 "초벌"입니다. 반드시 전문가 확인을 거치세요.

---

### 실제 워크플로우 예시

하나의 프로젝트를 AI 4개로 나눠서 처리하는 방법:

```
① 회의 녹음
   → NotebookLM (텍스트 추출 + 마인드맵)

② 마인드맵 기반으로 큰 그림 잡기
   → ChatGPT (구조 설계, 아이디어 확장)

③ 구조 확정 후 정밀 문서 작성
   → Claude (사업계획서, 제안서, 보고서)

④ 최종 검토
   → Perplexity (수치 팩트체크, 출처 확인)
```

이렇게 하면:
- 녹음 → 텍스트: **5분**
- 구조 설계: **15분**
- 문서 작성: **30분**
- 팩트체크: **10분**

총 **1시간**이면 사업계획서 초안이 나옵니다.
사람이 직접 하면 **이틀**이 걸릴 일입니다.

---

### 비용 대비 효과

> **"제일 비싼 걸로 돌리는 게 두 번 세 번 작업이 적어서 결국 싸게 먹혀요."**

| 접근 방식 | 비용 | 결과 |
|----------|------|------|
| 무료 모델만 사용 | ₩0 | 3번 돌려도 70점. 결국 직접 고침. 시간 3시간 추가. |
| 유료 모델 사용 | ₩27,000/월 | 1번에 90점. 약간만 수정. 시간 30분. |

**월 27,000원 아끼려다 매달 수십 시간을 낭비하고 있는 겁니다.**

여러분의 시급이 얼마든, 3시간이면 27,000원보다 비쌉니다.

## 4. 채팅은 휘발된다 — 결과를 저장하라

### 문제: 좋은 대화를 했는데 창을 닫으면 사라진다

AI와 30분 동안 좋은 대화를 했습니다.
사업계획서 구조가 딱 잡혔고, 핵심 메시지도 정했습니다.

다음 날 다시 열었습니다.
**AI는 다 잊었습니다.**

> **"채팅은 휘발됩니다. 창을 닫으면 AI는 다 잊습니다."**

물론 대부분의 AI 서비스가 대화 기록을 저장합니다.
하지만 그건 **기록**일 뿐, **정리된 결과물**이 아닙니다.
50개의 대화 기록 속에서 "그때 그 좋은 문장"을 찾는 건 거의 불가능합니다.

---

### 해결: 폴더 구조 만들기

프로젝트마다 이 구조를 만들어두세요:

```
내프로젝트/
├── raw/          ← 날것 메모, 녹음 텍스트, 아이디어
├── drafts/       ← AI와 작업한 초안들
├── final/        ← 확정된 결과물
└── rejected/     ← 거절한 것들 (나중에 쓸 수도 있음)
```

#### 각 폴더의 역할

| 폴더 | 뭘 넣나 | 예시 |
|------|---------|------|
| `raw/` | 생각의 날것. 정리 안 된 것들. | 녹음 텍스트, 메모, 스크린샷 |
| `drafts/` | AI와 주고받으며 만든 중간 결과물 | 초안 v1, v2, v3 |
| `final/` | 최종 확정된 것 | 제출한 사업계획서, 완성 보고서 |
| `rejected/` | 거절했지만 보관할 것 | "이건 아닌데" 싶었던 방향들 |

---

### 왜 rejected/ 가 중요한가

> **"거절한 내용도 다 모아두세요.**
> **'이건 아닌데'라는 판단 자체가 당신의 기준이니까요."**

여러분이 AI의 답을 거절할 때, 거기에는 **여러분의 판단**이 들어 있습니다.
"이건 너무 뻔해" → 여러분은 **뻔하지 않은 것**을 원한다는 뜻입니다.
"이 방향 아니야" → 여러분은 **다른 방향이 있다**는 뜻입니다.

이 거절의 모음이 쌓이면, 그게 바로 **여러분만의 기준**이 됩니다.
나중에 새 프로젝트를 할 때, rejected/ 폴더를 보면
**"내가 원하지 않는 것"**이 명확해집니다.

뭘 원하는지 모를 때, **뭘 원하지 않는지**부터 확인하는 겁니다.

---

### 저장 습관 체크리스트

```
□ AI와의 대화에서 좋은 결과가 나오면 즉시 파일로 저장한다
□ 파일 이름에 날짜를 넣는다 (예: 2026-06-19_사업계획_초안.md)
□ 거절한 것도 따로 모아둔다
□ 일주일에 한 번 폴더를 정리한다
```

### 저장하는 구체적인 방법

AI와 대화 중 좋은 결과가 나왔다면:

**방법 1: 복사 + 붙여넣기 (가장 기본)**
1. 좋은 답변을 드래그해서 복사
2. 메모장이나 문서에 붙여넣기
3. 폴더에 저장

**방법 2: AI에게 정리시키기 (추천)**
1. 대화가 어느 정도 진행되면
2. "지금까지 우리가 정한 것을 마크다운 문서로 정리해줘"
3. 정리된 결과를 파일로 저장

**방법 3: 전체 대화 내보내기**
1. ChatGPT: 설정 → 데이터 내보내기
2. Claude: 대화 내용 전체 복사
3. 이건 백업용. 평소에는 방법 2를 추천합니다.

## 5. 반복 작업을 찾아라

### 핵심 메시지

> **"재미없는 일을 AI한테 최대한 매뉴얼화해서 시켜야 돼요.**
> **그래야 우리가 더 재밌는 일을 많이 할 수 있어요."**

AI는 **창의적인 일**을 시키는 것보다
**반복적이고 지루한 일**을 시키는 게 효과가 큽니다.

왜? 반복 작업은:
- 매번 비슷한 패턴입니다
- 시간은 많이 들지만 판단은 적습니다
- 실수하면 안 되지만, 하기 싫은 일입니다

**이런 일이 AI에게 가장 잘 맞습니다.**

### 구체적으로 어떻게 하는가: 뉴스 스크랩 예시

**Before: 직접 하면**
1. 포털 사이트 접속 (2분)
2. "해양수산" 검색 (1분)
3. 기사 10개 읽기 (20분)
4. 중요한 것 5개 선별 (5분)
5. 요약 정리 (10분)

총: **약 38분**

**After: AI로 하면**

Perplexity에 이렇게 입력:
```
이번 주 해양수산 분야 주요 뉴스 5개를 아래 형식으로 정리해줘:
- 제목
- 핵심 내용 (2줄)
- 우리 사업과의 관련성 (1줄)
- 출처 링크
```

총: **약 3분**

**절약: 하루 35분 × 근무일 22일 = 월 약 13시간**

월 13시간이면 **이틀 치 근무**입니다.
AI 구독료 27,000원으로 이틀을 벌 수 있습니다.

## 6. 유튜브로 AI 쓰는 법을 배워라

### 가장 빠른 독학법

학원도, 책도 필요 없습니다. **유튜브에 다 있습니다.**

하지만 대부분은 이렇게 합니다:

> ❌ 영상 본다 → "오 좋다" → 다음 영상 본다 → 까먹는다

효과적인 방법은 이겁니다:

```
① 유튜브에서 좋은 강의를 찾는다
   ("claude md 설정", "AI 워크플로우", "프롬프트 팁" 등)
         ↓
② 텍스트로 추출한다
   (NotebookLM에 유튜브 링크 붙여넣기 → 자동 추출)
         ↓
③ AI에게 비교를 시킨다
   "이 강의 내용과 내가 지금 쓰는 방식을 비교해줘"
         ↓
④ 차이점만 내 것에 적용한다
   (통째로 복사하지 않는다!)
```

### 왜 이 방법이 좋은가

| 그냥 보기 | 추출 → 비교 → 적용 |
|----------|--------------------|
| 30분 시청 후 기억나는 것: 2~3개 | 텍스트로 남아서 언제든 검색 가능 |
| 남의 방법을 그대로 따라함 | 내 상황과 비교해서 맞는 것만 흡수 |
| 영상 다시 찾아야 함 | 정리된 텍스트가 내 폴더에 있음 |
| 적용하다 막히면 다시 시청 | AI에게 "이 부분 어떻게 적용해?" 가능 |

### 실전 예시: "Claude CLAUDE.md 설정법" 영상

**Step 1: NotebookLM에 유튜브 링크 넣기**

NotebookLM (notebook.google.com) 접속 → 소스 추가 → "YouTube" → 링크 붙여넣기

→ 자막이 자동으로 텍스트로 변환됩니다.

**Step 2: 핵심 추출 요청**

```
이 영상에서 말하는 CLAUDE.md 설정법의 핵심을 정리해줘:
1. 파일에 넣어야 하는 항목 목록
2. 각 항목의 예시
3. 하지 말라고 한 것
```

**Step 3: 내 것과 비교**

```
방금 정리한 설정법과 아래 내 현재 설정을 비교해줘.
차이점, 내가 빠뜨린 것, 추가하면 좋을 것을 알려줘.

[내 현재 설정]
- 역할: 부동산/해양 분야 VC 대표 보조
- 규칙: 수치에 근거 필수, 과장 금지
- 용어: SPC, LP, GP 정의
```

**Step 4: 적용**

AI가 "데이터 신선도 규칙이 빠져있다"고 알려주면 → 내 규칙에 추가:
```
- 통계/수치는 2년 이내 데이터만 사용. 오래된 데이터는 최신으로 리서치 후 사용.
```

### 추천 검색어

| 레벨 | 유튜브 검색어 | 배울 수 있는 것 |
|------|-------------|----------------|
| 초급 | "AI 프롬프트 잘 쓰는 법" | 기본 입력 구조 |
| 초급 | "NotebookLM 사용법" | 녹음/문서 정리 |
| 중급 | "claude md 설정" | 규칙 파일 작성 |
| 중급 | "AI custom instructions" | 맞춤 지침 설정 |
| 중급 | "AI 업무 자동화" | 반복 작업 위임 |
| 고급 | "claude code 워크플로우" | 로컬 시스템 구축 |
| 고급 | "MCP server 만들기" | 외부 도구 연결 |

> **핵심: 남의 설정을 그대로 쓰지 마세요.**
> 텍스트로 추출해서 내 것과 비교하고, 차이점만 흡수하세요.
> 그게 "독학"입니다.
