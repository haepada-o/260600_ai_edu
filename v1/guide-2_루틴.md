## 2-1. 루틴 분석 워크시트

AI를 도입하기 전에, 먼저 **내가 반복하고 있는 작업**을 파악해야 한다. 아래 표를 직접 채워 본다.

### 나의 반복 작업 목록

| # | 작업 | 빈도 | 소요시간 | 귀찮은 정도 (1~5) |
|---|------|------|---------|-------------------|
| 예시 | 촬영 견적서 작성 | 주 2회 | 1시간 | 4 |
| 예시 | SNS 캡션 작성 | 매일 | 30분 | 3 |
| 예시 | 미팅 후 정리 | 주 3회 | 40분 | 5 |
| 예시 | 레퍼런스 수집 | 수시 | 2시간 | 2 |
| 1 | | | | |
| 2 | | | | |
| 3 | | | | |
| 4 | | | | |
| 5 | | | | |
| 6 | | | | |
| 7 | | | | |
| 8 | | | | |

### 우선순위 공식

```
AI 전환 우선순위 = 빈도 × 소요시간 × 귀찮은 정도
```

**빈도 환산 기준:**

| 빈도 | 점수 |
|------|------|
| 매일 | 5 |
| 주 3~4회 | 4 |
| 주 1~2회 | 3 |
| 월 2~3회 | 2 |
| 월 1회 이하 | 1 |

**소요시간 환산 기준:**

| 소요시간 | 점수 |
|---------|------|
| 2시간 이상 | 5 |
| 1~2시간 | 4 |
| 30분~1시간 | 3 |
| 15~30분 | 2 |
| 15분 이하 | 1 |

### 계산 예시

| 작업 | 빈도 | 시간 | 귀찮음 | **점수** | **판정** |
|------|------|------|--------|---------|---------|
| 미팅 후 정리 | 4 | 3 | 5 | **60** | 🔴 즉시 전환 |
| 견적서 작성 | 3 | 4 | 4 | **48** | 🔴 즉시 전환 |
| SNS 캡션 | 5 | 2 | 3 | **30** | 🟡 전환 추천 |
| 레퍼런스 수집 | 3 | 5 | 2 | **30** | 🟡 전환 추천 |
| 이메일 답장 | 4 | 1 | 2 | **8** | 🟢 여유 있을 때 |

**판정 기준:**

| 점수 | 판정 | 의미 |
|------|------|------|
| 40 이상 | 🔴 즉시 전환 | 지금 당장 AI로 바꿔야 할 작업 |
| 20~39 | 🟡 전환 추천 | AI로 전환하면 체감 효과 큼 |
| 20 미만 | 🟢 여유 있을 때 | 수동으로 해도 괜찮음 |

> **점수 높은 것부터 하나씩 AI로 전환한다. 한 번에 다 바꾸지 않는다.**

## 2-2. 작업별 플랫폼 매칭

어떤 작업에 어떤 도구를 쓸지 한눈에 정리한 표.

### 전체 매칭표

| 작업 유형 | 추천 도구 | 왜 이 도구인가 |
|----------|----------|---------------|
| 견적서/제안서 작성 | **Claude** (Co-work) | 프로젝트별 맥락 유지, 양식 기억 |
| 회의/미팅 녹음 정리 | **NotebookLM** | 무료, 마인드맵 자동 생성, 요약 |
| SNS 캡션/해시태그 | **ChatGPT** | 톤 조절 유연, 여러 버전 생성 |
| 레퍼런스 이미지 생성 | **Midjourney** or **ChatGPT** | 퀄리티(MJ) vs 속도(ChatGPT) |
| 이메일 답장 | **Claude** | 맥락 유지, 톤 일관성 |
| 포트폴리오 정리 | **Claude** (Co-work) | 텍스트 구조화, 프로젝트별 관리 |
| 파일 정리/이름 변경 | **Cursor / Antigravity** | 일괄 처리, 스크립트 자동 생성 |
| 자막 생성 | **Whisper** (무료) | 정확도 높음, 오프라인 가능 |
| 컨셉 시각화 (빠르게) | **ChatGPT** | 대화 중 즉시 이미지 생성 |
| 컨셉 시각화 (정밀) | **ComfyUI** | 완전 제어, LoRA 적용 |
| 트렌드/밈 파악 | **Grok** | X(트위터) 실시간 데이터 연동 |
| 시장 리서치 | **Gemini** (Deep Research) | 검색 연동, 긴 보고서 생성 |
| 시놉시스/기획서 구조화 | **Claude** (Co-work) | 정밀 구조화, 맥락 유지 |
| 조명/촬영 세팅 기록 | **옵시디언** + **Claude** | 축적 → 검색 → 재활용 |

### 선택 기준 요약

```
"글" 쓸 때        → Claude
"그림" 그릴 때     → ChatGPT (빠르게) / Midjourney (예쁘게) / ComfyUI (정밀)
"정리" 할 때       → NotebookLM (녹음) / Claude (텍스트)
"검색" 할 때       → Gemini (깊게) / Grok (실시간)
"파일" 다룰 때     → Cursor / Antigravity
"기록" 할 때       → 옵시디언
```

### 무료 vs 유료 정리

| 도구 | 무료로 할 수 있는 것 | 유료 필요한 것 |
|------|-------------------|--------------|
| **ChatGPT** | 기본 대화, 이미지 제한적 생성 | 이미지 무제한, GPT-4o 풀 사용 ($20/월) |
| **Claude** | 기본 대화 (일일 제한 있음) | Co-work, 대용량 파일 ($20/월) |
| **Gemini** | Flash 모델 사용 | Deep Research ($20/월) |
| **NotebookLM** | 전 기능 무료 | — |
| **Grok** | X 프리미엄 포함 | X 프리미엄 가입 필요 |
| **Whisper** | 로컬 설치 무료 | API 사용 시 소액 |
| **ComfyUI** | 전 기능 무료 | GPU 하드웨어 필요 |
| **Midjourney** | — | $10~30/월 (무료 없음) |
| **옵시디언** | 전 기능 무료 | 동기화 ($4/월, 선택) |

## 2-3. 아티스트 직군별 루틴 예시

각 직군의 하루/워크플로우에 AI를 어디에 넣을 수 있는지 구체적으로 보여준다.

---

### 사진작가의 하루

```
[아침] 오늘 촬영 리스트 확인
       → Gemini: "오늘 촬영 [야외 인물] 날씨 [흐림]일 때 조명 세팅 추천"
       → 옵시디언: 이전 촬영 세팅 기록에서 유사 조건 검색

[촬영 전] 클라이언트 요청사항 정리
       → Claude (Co-work): 클라이언트 메일 붙여넣기
       → "촬영 체크리스트로 변환해줘"

[촬영 후] 셀렉 + 보정
       → Cursor/Antigravity: RAW 파일 날짜별 폴더 자동 분류
       → 파일명 일괄 변경 (프로젝트명_날짜_번호)

[저녁] SNS 업로드
       → ChatGPT: 사진 설명 + "인스타 캡션 3가지 버전"
       → 해시태그 자동 생성
```

#### 사진작가 — 작업별 도구 매핑

| 시간대 | 작업 | 도구 | 소요시간 (기존 → AI) |
|--------|------|------|---------------------|
| 아침 | 촬영 세팅 확인 | Gemini + 옵시디언 | 30분 → 5분 |
| 오전 | 요청사항 → 체크리스트 | Claude (Co-work) | 20분 → 3분 |
| 오후 | 파일 정리 + 이름 변경 | Cursor/Antigravity | 1시간 → 1분 (스크립트) |
| 저녁 | 캡션 + 해시태그 | ChatGPT | 20분 → 3분 |
| 저녁 | 견적서 작성 | Claude (Co-work) | 1시간 → 10분 |

---

### 영상제작자의 워크플로우

```
[기획] 시놉시스 작성
       → Claude (Co-work): 프로젝트 자료 올려놓기
       → "이걸 시놉시스 형식으로 구조화해줘"

[프리프로덕션] 콜시트/스케줄
       → Claude: "촬영 일정 3일, 장소 5곳, 스태프 8명"
       → 콜시트 자동 생성

[촬영] 현장 녹음 → 텍스트
       → NotebookLM: 현장 녹음 업로드 → 텍스트 추출 + 요약

[편집] 자막
       → Whisper: 대사 추출 → SRT 파일 생성

[납품] 보고서/결과물 정리
       → Claude (Co-work): 프로젝트 전체 맥락으로 결과 보고서
```

#### 영상제작자 — 단계별 도구 매핑

| 단계 | 작업 | 도구 | 소요시간 (기존 → AI) |
|------|------|------|---------------------|
| 기획 | 시놉시스 초안 | Claude (Co-work) | 반나절 → 1시간 |
| 프리프로덕션 | 콜시트 생성 | Claude | 2시간 → 15분 |
| 촬영 | 현장 녹음 정리 | NotebookLM | 1시간 → 5분 |
| 편집 | 자막 추출 | Whisper | 수작업 3시간 → 10분 |
| 납품 | 결과보고서 | Claude (Co-work) | 3시간 → 30분 |

---

### 공간기획자의 워크플로우

```
[리서치] 레퍼런스 수집
       → Gemini Deep Research: "2024-2025 몰입형 전시 사례 10선"
       → 결과를 옵시디언에 저장 + 태그

[기획] 공간 컨셉
       → Midjourney or ChatGPT: 컨셉 이미지 (무드보드)
       → ComfyUI: 정밀 시각화 (조명/공간감 제어)
       → Claude: 공간 기획서 구조화

[반복 세팅] 특정 공간 조명/장비 세팅
       → 조건별 세팅 기록을 옵시디언에 축적
       → "천장 5m + 면적 200평 + 프로젝터 3대" → 검색 → 이전 세팅 호출

[제안] 견적 + 제안서
       → Claude (Co-work): 제안서 초안 + 견적서 자동 생성
```

#### 공간기획자 — 단계별 도구 매핑

| 단계 | 작업 | 도구 | 소요시간 (기존 → AI) |
|------|------|------|---------------------|
| 리서치 | 사례 조사 | Gemini Deep Research | 이틀 → 2시간 |
| 기획 | 무드보드 | ChatGPT / Midjourney | 반나절 → 30분 |
| 기획 | 기획서 구조화 | Claude (Co-work) | 반나절 → 1시간 |
| 세팅 | 조건별 세팅 검색 | 옵시디언 | 기억 의존 → 즉시 검색 |
| 제안 | 제안서/견적서 | Claude (Co-work) | 반나절 → 1시간 |

---

### 직군별 핵심 도구 조합 요약

| 직군 | 메인 도구 1 | 메인 도구 2 | 보조 도구 |

## 2-4. 인스타에서 영감 → 아이디어 루틴화

### 문제

인스타 스크롤하며 "오 좋다" → 다음 날 잊음 → 같은 걸 또 찾음 → 쌓이지 않음.

### 루틴: 5단계

```
① 발견     인스타에서 영감이 오는 이미지/영상 발견
            → 스크린샷 or 저장

② 추출     ChatGPT or Gemini에 이미지 넣기
            → "이 이미지의 조명/색감/구도/분위기를 분석해줘"
            → "이 스타일을 키워드 5개로 정리해줘"

③ 저장     옵시디언에 기록
            → [[영감노트]] 2026-06-23
            → 태그: #조명 #로우앵글 #네온 #야간촬영
            → AI 분석 결과 붙여넣기

④ 연결     기존 메모와 링크
            → "이건 [[지난주 촬영]]의 조명과 비슷하네"
            → "[[프로젝트C]]에 이 분위기를 적용하면?"

⑤ 생성     축적된 영감 → AI에게 새 작업 요청
            → "내가 모은 영감노트 10개를 보고,
               공통 패턴을 찾아줘.
               이걸 기반으로 새 컨셉을 제안해줘."
```

### 주간 루틴

| 주기 | 할 일 | 소요시간 |
|------|-------|---------|
| **매일** | 인스타 스크롤 → 좋은 거 1~3개 스크린샷 → AI 분석 → 옵시디언 저장 | 5분 |
| **주 1회** | 이번 주 영감노트 모아보기 → 패턴 발견 → 다음 작업에 적용 | 15분 |
| **월 1회** | 한 달 영감 아카이브 → AI에게 "내 취향 분석해줘" | 30분 |

### 도구 체인

```
인스타 → 스크린샷 → ChatGPT (분석) → 옵시디언 (저장+연결) → Claude (활용)
```

### 영감노트 템플릿 (옵시디언용)

```markdown
# 영감노트 — 2026-06-23

## 출처
- 인스타 @계정명 / 핀터레스트 / 기타

## 스크린샷
![[스크린샷파일명.jpg]]

## AI 분석 결과
- 조명: 사이드 라이트, 자연광 + 반사판
- 색감: 따뜻한 톤, 살짝 탈색된 필름감
- 구도: 로우앵글, 인물 왼쪽 배치, 여백 우측
- 분위기: 고요함, 오후 늦은 시간

## 키워드
#골든아워 #로우앵글 #필름톤 #자연광 #인물

## 연결
- [[프로젝트C]] — 이 조명 참고
- [[야외촬영 세팅]] — 유사 조건

## 메모
- 반사판 각도가 핵심. 45도보다 낮게.
```

### 직군별 영감 수집 포인트

| 직군 | 수집할 것 | AI에게 물어볼 것 |
|------|----------|----------------|
| **사진작가** | 조명, 색감, 구도, 포즈 | "이 사진의 조명 세팅을 추정해줘" |
| **영상제작자** | 카메라 무빙, 컷 전환, 색보정 | "이 영상의 편집 패턴을 분석해줘" |
| **공간기획자** | 공간 구성, 동선, 소재 | "이 공간의 설계 원리를 분석해줘" |

## 2-5. 아이폰 자동화 — 녹음에서 문서까지

### 문제

미팅/현장에서 녹음 → 폰에 묻힘 → 안 씀 → 다음 미팅 때 "그때 뭐라고 했더라?"

### 목표 파이프라인

```
📱 아이폰 녹음
    ↓
☁️ 구글 드라이브 자동 업로드
    ↓
🔄 자동화 (Make/Zapier)
    ↓
📝 텍스트 추출 (Whisper)
    ↓
📄 문서화 (Claude → 옵시디언)
```

---

### 방법 A: 가장 현실적 (반자동)

> 설정 난이도: ★☆☆ | 비용: 무료 | 수동 조작: 10초 (공유 버튼 1번)

```
① 아이폰 음성메모로 녹음
② 녹음 끝 → 공유 버튼 → 구글 드라이브 선택 → 업로드
③ 구글 드라이브 특정 폴더에 저장됨
④ NotebookLM에서 해당 파일 열기 → 텍스트 추출 + 요약
⑤ 결과를 옵시디언에 저장
```

**장점:** 설정 없음, 무료, 지금 바로 가능
**단점:** 공유 버튼 누르는 동작은 수동

---

### 방법 B: 단축어 + 뒷면 탭 (반자동, 더 빠름)

> 설정 난이도: ★★☆ | 비용: 무료 | 수동 조작: 폰 뒷면 톡톡 2번

#### 사전 설정

```
아이폰 → 설정 → 손쉬운 사용 → 터치 → 뒷면 탭
→ "녹음 시작" 단축어 할당
```

#### 사용

```
① 폰 뒷면 톡톡 → 녹음 시작
② 다시 톡톡 → 녹음 종료 → 공유 → 드라이브 업로드

또는: 단축어로 녹음 → 파일 앱 저장 → 드라이브 자동 동기화
```

#### 단축어 설정 상세

| 단계 | 단축어 액션 | 설명 |
|------|-----------|------|
| 1 | 오디오 녹음 시작 | 녹음 시작 |
| 2 | 녹음 중지 (수동 or 타이머) | 녹음 파일 생성 |
| 3 | 파일 저장 (iCloud Drive or 구글 드라이브) | 특정 폴더에 저장 |
| 4 | (선택) 알림 보내기 | "녹음 저장 완료" 확인 |

---

### 방법 C: 완전 자동화 (Make/Zapier 연동)

> 설정 난이도: ★★★☆ | 비용: Make 무료 티어 가능 | 수동 조작: 녹음 시작/종료만

#### 파이프라인

```
📱 단축어로 녹음
    ↓
  녹음 파일을 Webhook URL로 전송 (단축어의 "URL 내용 가져오기")
    ↓
🔄 Make/Zapier가 수신
    ↓
  ① 구글 드라이브에 저장
  ② Whisper API로 텍스트 변환
  ③ Claude/ChatGPT API로 요약 + 구조화
    ↓
📝 결과를 구글 드라이브 or 노션 or 옵시디언에 저장
```

#### Make/Zapier 시나리오 상세

```
트리거:   Webhook (단축어에서 음성 파일 수신)
          또는: "구글 드라이브 특정 폴더에 새 파일 감지"

액션 1:   OpenAI Whisper → 텍스트 변환
액션 2:   ChatGPT/Claude → 요약 + 액션아이템 추출
액션 3:   구글 드라이브에 텍스트 파일 저장
          또는: 노션 DB에 행 추가
          또는: 이메일로 발송
```

#### Make 시나리오 흐름도

```
[Webhook 수신] → [Google Drive 업로드] → [OpenAI Whisper] → [ChatGPT 요약]
                                                                  ↓
                                          [노션 DB에 저장] ← [텍스트 가공]
```

---

### 방법 D: 전용 앱 (가장 간편)

> 설정 난이도: ★☆☆ | 비용: 앱 유료 (일부) | 수동 조작: 앱 실행만

```
Otter.ai / Audionotes / Clova Note
→ 앱 내에서 녹음 + 자동 텍스트 변환
→ Zapier 연동 → 구글 드라이브/노션에 자동 저장
```

| 앱 | 한국어 지원 | 가격 | 특징 |
|----|-----------|------|------|
| **Clova Note** | ✅ 우수 | 무료 (제한), 유료 | 네이버, 한국어 최적화 |
| **Otter.ai** | ⚠️ 영어 중심 | $8.33/월~ | 영어 정확도 최고, Zapier 연동 |
| **Audionotes** | ⚠️ 제한적 | $10/월~ | AI 요약 내장 |

**장점:** 설정 간단, 바로 사용 가능
**단점:** 앱 유료 (일부), 한국어 정확도 차이 있음

## 2-6. 파일/데이터 관리 자동화

### 아티스트의 현실

```
SD카드 → 바탕화면 → 외장하드 → 폰 → 드라이브 → 어디?

"그 사진 어디 있지?"     → 30분 찾기
"작년 프로젝트 파일은?"   → 못 찾음
"이 영상 원본이 어딨어?"  → 3개 기기에 흩어져 있음
"파일 이름이 IMG_4392..."  → 뭔지 모름
```

### 문제 진단표

| 문제 | 증상 | 심각도 |
|------|------|--------|
| **이름 규칙 없음** | IMG_4392.jpg, 제목없음.mp4, 최종_최종_진짜최종.psd | 🔴 |
| **폴더 규칙 없음** | 바탕화면에 300개 파일, "새 폴더 (3)" | 🔴 |
| **버전 관리 없음** | 최종, 최종2, 진짜최종, 수정본_최종 | 🟡 |
| **백업 없음** | 외장하드 고장 → 3년치 작업물 소실 | 🔴 |

### 해결 1: 파일 이름 자동 변경

#### Before → After

```
Before:                              After:
  IMG_4392.jpg                        2026-06-23_프로젝트A_인물_001.jpg
  IMG_4393.jpg                        2026-06-23_프로젝트A_인물_002.jpg
  IMG_4394.jpg                        2026-06-23_프로젝트A_인물_003.jpg
  DSC_0001.ARW                        2026-06-23_프로젝트A_인물_001.ARW
  MVI_0023.MP4                        2026-06-23_프로젝트A_비하인드_001.MP4
```

#### 방법: Cursor/Antigravity에게 요청

```
"이 폴더에 있는 사진 파일 이름을
 [날짜]_[프로젝트명]_[유형]_[번호] 형식으로 바꿔줘.
 EXIF 데이터에서 날짜 추출해서 사용해."

→ 스크립트 자동 생성 → 실행 → 1000장 10초
```

#### 이름 규칙 추천

| 항목 | 형식 | 예시 |
|------|------|------|
| 날짜 | YYYY-MM-DD | 2026-06-23 |
| 프로젝트명 | 한글 or 영문 | 전시A, brandX |
| 유형 | 인물, 제품, 공간, 비하인드 | 인물 |
| 번호 | 3자리 | 001, 002 |
| 구분자 | 언더스코어 _ | _ |
| **전체** | **날짜_프로젝트_유형_번호** | **2026-06-23_전시A_인물_001.jpg** |

> **언더스코어(_)를 쓰는 이유:** 하이픈(-)은 날짜에 쓰이므로, 항목 구분에는 언더스코어를 사용. 공백은 프로그래밍 시 문제를 일으키므로 절대 금지.

#### 표준 구조 템플릿

```

"새 프로젝트 폴더 만들어줘. 이름: 2026_전시A"

→ 자동 생성:

│   ├── raw/

├── 04_편집/

│   ├── v2/

### 해결 3: 파일 자동 분류

→ 실행:
  사진 (JPG, RAW, PNG)  → /photos/
  영상 (MP4, MOV)       → /videos/
  문서 (PDF, DOCX)      → /docs/

```

```
"EXIF 날짜 기준으로 월별 폴더에 넣어줘"

→ 실행:
  /2026-01/
  /2026-02/
  /2026-03/
  ...
```

#### 분류 방식 비교

| 분류 기준 | 적합한 상황 | 명령어 예시 |
|----------|-----------|-----------|
| 확장자별 | 여러 종류 파일이 섞여 있을 때 | "확장자별로 분류해줘" |
| 날짜별 | 한 종류 파일이 대량일 때 (사진) | "EXIF 날짜 기준 월별 분류" |
| 프로젝트별 | 파일명에 프로젝트명 포함 시 | "파일명의 프로젝트명으로 분류" |
| 크기별 | 대용량 파일 정리 시 | "1GB 이상 파일만 따로 모아줘" |

### 해결 4: 버전 관리 규칙

#### 잘못된 방법 vs 올바른 방법

```
❌ 이름으로 버전 관리하지 않기:
  최종.psd
  최종_수정.psd
  최종_수정_진짜.psd
  최종_수정_진짜_이거_찐.psd

✅ 날짜 + 버전 번호:
  2026-06-23_전시A_포스터_v1.psd
  2026-06-23_전시A_포스터_v2.psd
  2026-06-24_전시A_포스터_v3_final.psd
```

#### 버전 관리 규칙

| 규칙 | 설명 | 예시 |
|------|------|------|
| v1, v2, v3 | 내부 수정 버전 | _v1, _v2, _v3 |
| _final | 클라이언트 납품 확정본 | _v3_final |
| _rev1 | 클라이언트 수정 요청 반영 | _v3_final_rev1 |
| 날짜 항상 포함 | 언제 만들었는지 즉시 파악 | 2026-06-24_ |

#### AI로 자동 적용

```
"이 폴더에 있는 파일 중 같은 프로젝트의 파일들을
 날짜_프로젝트_파일명_v번호 형식으로 정리해줘"
```

### 해결 5: 백업 자동화

#### 백업 구조

```
최소 구조 (무료):
  원본 → 구글 드라이브 (자동 동기화)

권장 구조 (3-2-1 백업):
  원본     → 작업 컴퓨터
  백업 1   → 외장하드 (주 1회)
  백업 2   → 구글 드라이브 or iCloud (자동 동기화)
```

**3-2-1 백업 규칙:**

| 숫자 | 의미 | 실천 |
|------|------|------|
| **3** | 최소 3개의 복사본 | 컴퓨터 + 외장하드 + 클라우드 |
| **2** | 2가지 다른 저장 매체 | SSD + 클라우드 |
| **1** | 1개는 물리적으로 다른 장소 | 클라우드 (화재/도난 대비) |

#### 구글 드라이브 자동 동기화 설정

```
1. Google Drive for Desktop 설치
2. 작업 폴더를 동기화 대상으로 지정
3. 끝. 파일 저장하면 자동으로 클라우드 백업
```

#### 클라우드 저장소 비교

| 서비스 | 무료 용량 | 유료 가격 | 특징 |
|--------|----------|----------|------|

## 2-7. 마크다운(.md) — AI 시대의 기본 데이터 포맷

### 왜 .md인가

| 포맷 | AI가 읽을 때 | 토큰 소비 | 편집 | 버전 관리 |
|------|-------------|----------|------|----------|
| **HWP** | 읽을 수는 있음. 변환 과정에서 표/서식 깨짐 다수 | 🔴 **매우 높음** — 서식 메타데이터가 토큰을 잡아먹음 | 한글에서만 | 불가 |
| **PDF** | 읽을 수 있음. 이미지 기반이면 OCR 필요 | 🔴 **높음** — 레이아웃 정보가 토큰에 포함 | 사실상 불가 | 불가 |
| **DOCX** | 읽을 수 있음 | 🟡 보통 | Word 필요 | 어려움 |
| **.md (마크다운)** | **네이티브** — AI가 가장 잘 읽고 쓰는 포맷 | 🟢 **최소** — 내용만 있음 | 메모장으로도 가능 | Git 가능 |

```
같은 내용의 견적서:
  HWP  → AI에게 줄 때 ~3,000 토큰 (서식 포함)
  PDF  → AI에게 줄 때 ~2,500 토큰 (레이아웃 포함)
  .md  → AI에게 줄 때 ~800 토큰 (내용만)

= 같은 돈으로 3배 더 많이 시킬 수 있다
```

### 마크다운 = 메모장 + 약간의 규칙

```markdown
# 제목 (큰 글씨)
## 소제목
### 소소제목

**굵게**
*기울임*

- 목록 1
- 목록 2

| 항목 | 내용 |
|------|------|
| A    | 설명 |

> 인용문

```코드블록```
```

**이게 전부다.** 5분이면 배운다.
옵시디언, 노션, GitHub, Claude, Cursor — 전부 .md를 기본으로 쓴다.

### 기존 데이터 → .md 변환 방법

```
[방법 1] AI에게 변환 시키기 (가장 쉬움)

  HWP/PDF 파일을 Claude나 ChatGPT에 업로드
  → "이 내용을 마크다운 형식으로 변환해줘"
  → .md 파일로 저장

[방법 2] NotebookLM 활용

  HWP/PDF를 NotebookLM에 업로드
  → 텍스트 추출 + 요약
  → 결과를 .md로 저장

[방법 3] Cursor/Antigravity에서 일괄 변환

  "이 폴더에 있는 PDF 10개를 전부 .md로 변환해줘"
  → 스크립트 자동 생성 → 실행
```

**우선순위**: 모든 문서를 한 번에 바꿀 필요 없음
```
1순위: 반복 사용하는 문서 (견적서 템플릿, 체크리스트, 제안서 양식)
2순위: 기준 문서 (심사 기준표, 브랜드 가이드, 작업 규칙)
3순위: 아카이브 (과거 프로젝트 자료)
```

### .md의 진짜 힘 — "살아있는 기준 문서"

**기획자 관점에서 가장 중요한 사용법:**

```
기존 방식:
  기준 문서가 여러 곳에 흩어짐
  → HWP 3개, PDF 2개, 엑셀 1개, 메일 본문 어딘가
  → 검증할 때마다 "그거 어디 있더라?" → 30분 찾기
  → 버전이 안 맞음 → 사고

.md 방식:
  기준 문서 1개에 통합 → 계속 업데이트
  → AI에게 통째로 줌 → "이 기준에 맞는지 검증해줘"
```

**실전 예시: 공모전 제안서 검증 기준**

```markdown
# 제안서 검증 기준.md

## 공고 요구사항 (이 공고에서 추출)
- 사업 기간: 2026.09 ~ 2027.08
- 예산 상한: 5,000만원
- 필수 항목: 사업계획서, 예산서, 참여인력 경력

## 배점 기준 (공고에서 추출)
| 항목 | 배점 | 주의 |
|------|------|------|
| 사업 이해도 | 20 | "구체적 실행방안" 명시 요구 |
| 수행 역량 | 30 | 유사 사업 실적 중심 |
| 예산 적정성 | 20 | 항목별 근거 필수 |
| 기대 효과 | 15 | 정량 지표 선호 |
| 지역 연계 | 15 | 로컬 파트너 유무 |

## 우리 팀 강점 매칭
- 수행 역량 → 작년 유사 프로젝트 3건
- 지역 연계 → 부산 파트너 확보

## 금지 사항
- 예산에 인건비 50% 초과 금지
- "혁신적" "선도적" 등 추상 표현 금지

## 검증 체크리스트
- [ ] 모든 필수 항목 포함?
- [ ] 배점 30점 항목(수행역량) 충분히 서술?
- [ ] 예산 항목별 근거 명시?
- [ ] 금지 사항 위반 없음?
```

**이 파일 하나로 할 수 있는 것**:
```
① "이 기준 문서를 읽고, 내 제안서 초안을 검증해줘"
   → AI가 항목별로 통과/미달 체크

② "배점 30점짜리(수행 역량)가 약한데, 보강할 내용 제안해줘"
   → AI가 기준에 맞춰 제안

③ 다음 공모전에서 공고만 바꿔 넣으면 → 기준 문서 재활용
   → 매번 처음부터 만들 필요 없음
```

### 아티스트 적용: .md로 만들 기준 문서들

| 직군 | 기준 문서 | 뭘 넣나 |
|------|----------|--------|
| **사진작가** | `촬영기준.md` | 조건별 세팅, 납품 규격, 보정 기준, 클라이언트 요구사항 |
| **영상제작자** | `제작기준.md` | 시놉시스 양식, 콜시트 형식, 납품 포맷, 자막 규칙 |
| **공간기획자** | `전시기준.md` | 공간 조건 체크리스트, 장비 목록, 안전 기준, 예산 항목 |
| **댄서** | `공연기준.md` | 음악 BPM 범위, 조명 큐시트 형식, 리허설 체크리스트 |

**사진작가 예시: 촬영기준.md**

```markdown
# 촬영 기준

## 납품 규격
- 포맷: JPEG (최종) + RAW (원본)
- 해상도: 최소 4000px 장축
- 색공간: sRGB (웹용) / Adobe RGB (인쇄용)
- 파일명: YYYYMMDD_프로젝트_번호.jpg

## 보정 기준
- 화이트밸런스: 반드시 보정
- 피부 보정: 과도한 스무딩 금지, 결 유지
- 배경 제거: 요청 시만
- 컬러그레이딩: 프로젝트별 LUT 적용 (있을 경우)

## 조건별 기본 세팅
| 조건 | ISO | 조리개 | 셔터 | 보조장비 |
|------|-----|--------|------|---------|
| 야외 맑음 | 100-200 | f/2.8-4 | 1/500+ | 반사판 |
| 야외 흐림 | 400-800 | f/2.8 | 1/250 | 반사판 + 스트로브 |
| 실내 자연광 | 800-1600 | f/1.8-2.8 | 1/125 | 반사판 |
| 스튜디오 | 100 | f/8-11 | 1/160 | 스트로브 3점 |

## 검증 체크리스트 (납품 전)
- [ ] 컬러 프로파일 확인 (sRGB/Adobe RGB)
- [ ] 파일명 규칙 준수
- [ ] 클라이언트 요구사항 전부 반영
- [ ] 셀렉 수량 맞음
```

**이렇게 만들어두면**:
```
"이 촬영기준.md 를 기반으로 오늘 촬영 체크리스트 만들어줘.
 조건: 야외 흐림, 인물 전신, 클라이언트 요구사항 [붙여넣기]"

→ AI가 기준 문서에서 세팅 자동 매칭 + 클라이언트 요구 반영 + 체크리스트 생성
```

### 핵심 정리

```
1. AI에게 줄 데이터는 .md가 기본
   → HWP/PDF는 토큰 낭비, 서식 깨짐
   → 같은 돈으로 3배 더 일시킬 수 있다

2. 반복 사용하는 문서부터 .md로 변환
   → 견적서, 체크리스트, 제안서 양식

3. "기준 문서"를 .md로 만들어 유지
   → 여러 곳에 흩어진 기준을 하나로 통합
   → AI에게 통째로 줘서 검증 기준으로 활용
   → 업데이트할 때 이 파일 하나만 수정

4. 기준 문서 = 계속 진화하는 살아있는 문서
   → 프로젝트마다 학습한 것을 추가
   → 시간이 갈수록 정밀해짐
```
