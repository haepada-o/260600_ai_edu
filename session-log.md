### 핵심 교훈

#### 1. 참석자 수준을 과대평가했다

```
가정: "AI로 대화는 해봤고, 시스템을 만들고 싶다"
실제: "모델 선택이 뭔지 모르겠다" / "결제 아무것도 안 했다"
```

→ **사전 설문 필수**. 5분이면 레벨 0/1/2/3 분류 가능.

#### 2. 감정이 먼저다

- 기범: "어제 형이 한 설명이 전혀 이해 안 갔다" → 당일 오전까지 감정 남아있음
- 대표님: AI에 대한 좌절감 → 화를 AI에게 전이
- 교훈: **감정 처리 10분을 공식 아젠다에 넣어야 함**

#### 3. "보여주기" > "설명하기"

기범의 정확한 요청:
> "형보다는 스승님이 뭘 원래 주시니 제가 질문을 하면서 형성이 되게"

→ 구조화된 슬라이드 설명이 아니라 **라이브 시연 + 질문**이 효과적
→ 프레젠테이션은 사후 배포 레퍼런스로 전환

#### 4. 한 문장이 32장보다 강하다

실제로 대표님의 문제 50%를 해결한 한 문장:
> **"높은 모델 쓰세요."**

이 한 문장이 시스템 설계 강의 전체보다 임팩트가 컸음.

#### 5. 수준별 즉석 분기가 필요하다

같은 시간에 3명에게 완전히 다른 강의를 함:
| 참석자 | 수준 | 실제 가르친 것 |
|--------|------|---------------|
| 대표님 | 레벨 0 | 모델 선택, 데이터 관리, NotebookLM |
| 기범 | 레벨 -1 | 감정 처리 → 화면 보여주기 |
| 3번째 | 레벨 2 | Ollama 설치, 로컬 LLM, MCP |

→ **3인 이상이면 1:1 과외가 아니라 사전 그루핑 필요**

#### 6. "제일 싼 거" 프레이밍이 먹힌다

```
"녹음 파일은 여기 올려서 하는게 제일 싸게 쳐요"
"제일 비싼 걸로 돌리는게 두 번 세 번 작업이 적어서"
```

→ 감성이 아니라 **비용/효율 프레임**으로 설득

## 내가 실제로 AI를 배운 방법

> 독학의 핵심 루프. 교육 자료에도 이 방법을 반영해야 한다.

### 유튜브 → 텍스트 추출 → 내 시스템에 적용

```
① 유튜브에서 좋은 강의 발견
   (skill 설정, claude.md 작성법, memory 관리 등)
        ↓
② 텍스트 추출
   (NotebookLM / Whisper / 자막 추출)
        ↓
③ 내 기존 시스템과 비교
   (이미 설정된 CLAUDE.md, agents/, skills/ 등 파악)
        ↓
④ 차이점 기반 개선 적용
   (부족한 부분 추가, 구조 변경, 새 패턴 도입)
        ↓
⑤ 실제 사용하면서 검증
```

### 이 방법이 효과적인 이유

- **남의 시스템을 통째로 복사하지 않는다** — 내 시스템과 비교해서 차이점만 흡수
- **텍스트로 추출하기 때문에 AI에게 바로 비교 요청 가능** — "이 강의 내용과 내 CLAUDE.md를 비교해서 개선점 찾아줘"
- **영상을 반복 안 봐도 된다** — 텍스트로 남기니 검색·참조 가능

### 교육 자료 반영 포인트

레벨 1~2 문서에 이 방법을 추가할 것:
- "유튜브에서 배우는 방법" 섹션
- "남의 설정을 그대로 쓰지 말고, 비교해서 내 것에 녹여라"
- NotebookLM에 유튜브 링크 넣으면 자막 자동 추출 → 정리 가능

## 콘텐츠 전략 — "강의 말고 과외를 찍어라"

> 2026-06-19 대화에서 도출

### 녹취록에서 발견된 5가지 "팔리는 순간"

1. **"아씨" 순간** — 모델 바꾸니 결과가 달라짐 (3초 반전)
2. **"대박이다" 순간** — NotebookLM 마인드맵 눈앞에서 생성
3. **"화내면 안 돼" 순간** — 불편한 진실 직접 말함
4. **"따로 합시다" 순간** — 감정을 자르고 다음 주제로
5. **취약함 공유** — "존나 해봤더라고요" 고백

### 최적 포맷

| 순위 | 포맷 | 핵심 |
|------|------|------|
| 1 | **"AI 과외" 유튜브 시리즈** | 실제 과외를 촬영. 날것이 강점. |
| 2 | **"3초 반전" 숏폼** | "아씨 순간"을 60초로 |
| 3 | **"실물 공개" 롱폼** | 실제 시스템 화면 녹화 설명 |

### 핵심 인사이트

> 당신의 화법은 "설명"이 아니라 **"진단→처방→시연"**이다.
> 이건 강의 포맷이 아니라 **리얼리티 포맷**이다.
> 백종원의 골목식당이 요리 강의가 아닌 것처럼.
