## 1-1. 언어모델 비교

### 주요 언어모델 한눈에 보기

| | **ChatGPT** | **Claude** | **Gemini** | **Grok** | **Llama** |
|---|---|---|---|---|---|
| **만든 곳** | OpenAI | Anthropic | Google | xAI | Meta |
| **잘하는 것** | 이미지 생성, 범용 대화 | 정밀 문서, 코드, 구조화 | 검색 연동, 긴 맥락, 멀티모달 | 실시간 정보, 제한 없는 톤 | 로컬 설치, 커스터마이징 |
| **못하는 것** | 긴 문서 구조화 | 이미지 생성 | 정밀 작업 | 정밀 문서 | 설치 난이도 |
| **가격** | $20/월 | $20/월 | $20/월 | X 프리미엄 포함 | 무료 (로컬) |

| **특이점** | 이미지 생성 내장 | 프로젝트/아티팩트 기능 | NotebookLM 연동 | X(트위터) 실시간 데이터 | 내 컴퓨터에서 돌림, 데이터 외부 유출 0 |

### + NotebookLM (별도 도구)

NotebookLM은 언어모델이 아니라 **정리 도구**다.

| 항목 | 내용 |
|------|------|
| **만든 곳** | Google |
| **가격** | 무료 |
| **핵심 기능** | 녹음/문서 업로드 → 텍스트 추출 + 마인드맵 + 요약 |
| **쓸 때** | 미팅 녹음 정리, 긴 문서 요약, 레퍼런스 분석 |
| **쓰는 법** | notebooklm.google.com → 자료 업로드 → 질문 |

```
실전 예시:
  클라이언트 미팅 1시간 녹음 → NotebookLM 업로드
  → "핵심 요구사항 3가지로 정리해줘"
  → "액션 아이템 뽑아줘"
  → 마인드맵으로 전체 구조 시각화
```

---

### Grok은 언제 쓰나

| 상황 | 왜 Grok인가 |
|------|------------|
| 최신 트렌드/밈 파악 | X(트위터) 실시간 데이터 접근 |
| 다른 AI가 거부하는 질문 | 표현 제한이 적음 |
| 빠른 레퍼런스 서치 | 실시간 검색 내장 |
| 업계 반응 탐색 | X 게시물 분석 가능 |

```
실전 예시:
  "최근 1주일간 X에서 '몰입형 전시'에 대한 반응 정리해줘"
  "요즘 인스타에서 유행하는 사진 스타일 트렌드 알려줘"
  "이 아티스트에 대한 최근 언급 모아줘"
```

---

### Llama는 언제 쓰나

| 상황 | 왜 Llama인가 |
|------|-------------|
| 민감한 데이터를 외부에 보내기 싫을 때 | 완전 로컬 — 데이터가 내 컴퓨터 밖으로 안 나감 |
| 커스텀 모델을 만들고 싶을 때 | 오픈소스 — 원하는 대로 수정 가능 |
| 인터넷 없이 써야 할 때 | 오프라인 동작 |

```
설치 방법 (간소화):
  ① Ollama 설치 (ollama.com)
  ② 터미널에서: ollama run llama3
  ③ 끝. 대화 가능.

주의: 
  - GPU가 있으면 빠르고, 없으면 느림
  - 설치에 기술적 지식 약간 필요
  - 성능은 유료 서비스보다 낮을 수 있음
```

## 1-2. 모델 등급 차이

### 모든 AI에는 3개 등급이 있다

같은 회사 제품이라도 모델 등급이 다르다. 가격, 속도, 품질이 다 다르다.

```
Fast (빠른 모델)
├── 용도: 단순 질문, 번역, 요약
├── 예: GPT-4o mini, Claude Haiku, Gemini Flash, Llama 3 8B
└── 비유: 인턴 — 빠르지만 얕다

Standard (기본 모델)
├── 용도: 일반 대화, 초안 작성
├── 예: GPT-4o, Claude Sonnet, Gemini Pro, Grok-2, Llama 3 70B
└── 비유: 대리 — 무난하다

Deep (깊은 모델)
├── 용도: 복잡한 분석, 정밀 문서, 코드
├── 예: o3, Claude Opus, Gemini Deep Research, Llama 3 405B
└── 비유: 팀장 — 느리지만 깊다
```

### 등급별 비교표

| | **Fast** | **Standard** | **Deep** |
|---|---|---|---|
| **속도** | ★★★★★ | ★★★☆☆ | ★★☆☆☆ |
| **품질** | ★★☆☆☆ | ★★★☆☆ | ★★★★★ |
| **비용** | 최저 | 중간 | 최고 (사용량 많이 소모) |
| **적합한 작업** | 번역, 요약, 단순 질문 | 초안, 일반 대화, 브레인스토밍 | 제안서, 분석, 정밀 문서, 코드 |

### 모델별 등급 대응표

| 등급 | ChatGPT (OpenAI) | Claude (Anthropic) | Gemini (Google) | Llama (Meta) |
|------|---|---|---|---|
| **Fast** | GPT-4o mini | Haiku | Flash | 8B |
| **Standard** | GPT-4o | Sonnet | Pro | 70B |
| **Deep** | o3 | Opus | Deep Research | 405B |

### 언제 어떤 등급을 쓸 것인가

| 작업 | 등급 | 이유 |
|------|------|------|
| "이거 영어로 번역해줘" | Fast | 단순 변환, 빠르면 됨 |
| "인스타 캡션 써줘" | Fast~Standard | 간단한 텍스트 생성 |
| "이 촬영 컨셉 아이디어 줘" | Standard | 창의적이되 복잡하지 않음 |
| "이 기획서 구조 잡아줘" | Standard | 구조화 필요 |
| "공모전 제안서 써줘" | **Deep** | 결과물 품질이 곧 당락 |
| "이 계약서 분석해줘" | **Deep** | 정밀함이 필수 |
| "시놉시스 최종본 다듬어줘" | **Deep** | 완성도가 돈과 직결 |

> **원칙: 결과가 중요한 작업 → 무조건 Deep.**
> Fast로 제안서 쓰면 Fast 수준의 결과가 나온다. 싼 게 비지떡.

## 1-3. 웹 vs 설치형

### 같은 AI인데 사용법이 다른 것

| | **웹 (SaaS)** | **설치형 (에디터)** |
|---|---|---|
| **뭐냐** | 브라우저에서 대화 | 내 컴퓨터에 설치, 파일 직접 조작 |
| **예시** | claude.ai, chatgpt.com, gemini.google.com | **Cursor**, **Antigravity** (같은 종류), **Claude Code** (터미널) |
| **비유** | 전화 상담 | 직원이 내 사무실에 와서 직접 일함 |
| **언제 쓰나** | 아이디어, 초안, 질문, 대화 | 반복 자동화, 웹사이트, 데이터 처리, 파일 조작 |
| **난이도** | ★☆☆ | ★★☆ |
| **파일 접근** | 수동 업로드만 | 폴더 전체 자동 읽기/수정 |
| **외부 도구 연결** | ❌ 불가 | ✅ MCP로 가능 |

> **Cursor = Antigravity = 같은 종류.** 둘 다 "AI가 내 파일을 직접 읽고 수정하는 에디터".
> **Claude Code**도 본질은 같음. 다만 에디터 화면 대신 터미널(텍스트 명령창)에서 동작.

---

### 아티스트가 설치형을 써야 하는 순간

| 작업 | 웹으로 가능? | 설치형이 나은 이유 |
|------|------------|-----------------|
| 포트폴리오 웹사이트 직접 만들기 | ❌ | 파일 생성/수정을 직접 해줌 |
| 사진 파일 1000장 이름 일괄 변경 | ❌ | 스크립트 자동 실행 |
| 촬영 스케줄 자동 생성 | △ 수동 복붙 | 템플릿 기반 자동 |
| 인스타 캡션 100개 한 번에 | △ 느림 | 파일로 일괄 출력 |
| 공간 데이터 시각화 | ❌ | 데이터 읽기 + 차트 생성 |
| 반복하는 모든 것 | △ 수동 | 자동화 |

---

### 설치형의 진짜 힘 = 확장(Extension)

```
웹 AI:     대화만 가능 (복사 → 붙여넣기 → 복사 → 붙여넣기)
설치형 AI:  내 파일을 직접 조작 + 외부 도구와 연결 가능

            ┌─ 내 파일 시스템
설치형 ─────┼─ 구글 드라이브
            ├─ 노션
            ├─ 피그마
            ├─ 블렌더
            └─ 무엇이든 (MCP로 연결)

= AI가 "대화 상대"에서 "작업 파트너"로 바뀌는 지점
```

### AI 활용의 단계별 확장

```
1단계: 대화      → 웹에서 질문/답변 (누구나 여기서 시작)
2단계: 파일 조작  → 설치형으로 내 파일 직접 수정 (Cursor/Antigravity)
3단계: 도구 연결  → MCP로 외부 서비스 연결 (구글드라이브, 노션, 피그마)
4단계: 지식 축적  → 옵시디언 + LLM 위키 (시간이 쌓일수록 강해짐)
5단계: 시스템화   → 규칙 파일 + 에이전트 + 파이프라인 (자동화)

각 단계는 이전 단계를 포함한다.
5단계에 있는 사람은 1~4를 전부 쓰고 있다.
```

### 설치형 도구 비교

| | **Cursor** | **Antigravity** | **Claude Code** |
|---|---|---|---|
| **화면** | 에디터 (VS Code 기반) | 에디터 (VS Code 기반) | 터미널 (텍스트만) |
| **AI 엔진** | 여러 모델 선택 가능 | Claude 기본 | Claude 전용 |
| **가격** | $20/월~ | $20/월~ | Claude 요금에 포함 |
| **난이도** | ★★☆ | ★★☆ | ★★★ |
| **MCP 지원** | ✅ | ✅ | ✅ |
| **추천 대상** | 다양한 모델 쓰고 싶은 사람 | Claude 선호하는 사람 | 터미널에 익숙한 사람 |

## 1-4. Claude 심화

### 같은 Claude인데 쓰는 방식이 3개

| | **Chat** | **Co-work (Projects)** | **Code** |
|---|---|---|---|
| **어디서** | claude.ai 기본 화면 | claude.ai → Projects | 터미널 (`claude` 명령어) |
| **뭘 하나** | 1회성 대화 | 프로젝트별 맥락 유지 + 파일 공유 | 내 컴퓨터 파일 직접 수정 |
| **맥락 유지** | ❌ 창 닫으면 끝 | ✅ 프로젝트 안에서 유지 | ✅ 폴더 전체를 읽음 |
| **파일 업로드** | 1회성 | 프로젝트에 고정 | 자동 (폴더 내 전부) |
| **비유** | 카페에서 잠깐 상담 | 담당 직원 배정 | 직원이 내 사무실에 상주 |

---

### 핵심 차이: 맥락을 기억하느냐

```
Chat:     매번 처음부터 설명해야 함
Co-work:  "이 프로젝트에 대해 알고 있지?" → 알고 있음
Code:     "이 폴더에 있는 파일 다 봐" → 다 봄
```

---

### 각 모드 실전 사용법

#### Chat 모드 — 1회성 질문

```
쓸 때:
  "이거 번역해줘"
  "아이디어 좀 줘"
  "이 텍스트 요약해줘"
  "이 사진 분석해줘" (이미지 업로드)

특징:
  - 빠르다
  - 새 대화 열면 이전 내용 모름
  - 맥락 쌓을 필요 없는 단발 작업용
```

#### Co-work (Projects) 모드 — 프로젝트 단위 작업

```
쓸 때:
  프로젝트별로 자료 올려놓고 지속 작업

설정 방법:
  ① claude.ai → 왼쪽 패널 → "Projects"
  ② 새 프로젝트 생성 (예: "2026 전시A")
  ③ 프로젝트 설명 입력 (= 규칙 파일 역할)
  ④ 관련 파일 업로드 (기획서, 레퍼런스, 회의록)
  ⑤ 이후 이 프로젝트 안에서 대화하면 → 업로드한 자료를 전부 참고

실전 예시:
  "전시A" 프로젝트에 기획서, 레퍼런스, 회의록 올려놓기
  → "지금까지 올린 자료 기반으로 제안서 써줘"
  → "지난 회의에서 클라이언트가 말한 핵심 요구사항 뽑아줘"
  → "이 기획에서 빠진 부분 지적해줘"
```

#### Code 모드 — 파일 직접 조작

```
쓸 때:
  파일 자동화, 웹사이트 제작, 코드 실행, 일괄 처리

실행 방법:
  ① 터미널 열기
  ② 작업할 폴더로 이동
  ③ claude 입력 → 대화 시작
  ④ 해당 폴더의 모든 파일을 자동으로 읽음

실전 예시:
  "이 폴더에 있는 사진 파일 1000장 이름을
   날짜_프로젝트_번호 형식으로 바꿔줘"
  → 스크립트 작성 → 실행 → 10초에 끝
```

### 아티스트별 Claude 모드 추천

| 직군 | Chat | Co-work | Code |
|------|------|---------|------|
| **사진작가** | 촬영 아이디어, 캡션 | 클라이언트별 프로젝트 관리 | 파일 이름 일괄 변경 |
| **영상제작자** | 빠른 질문 | 시놉시스/기획서 (자료 축적) | 자막 일괄 생성 |
| **댄서** | 안무 아이디어 | 공연 기획 프로젝트 | 음악 BPM 분석 |
| **기획자** | 번역, 요약 | 공모전/프로젝트별 제안서 | 문서 일괄 처리 |
| **공간기획자** | 레퍼런스 분석 | 공간별 기획서 | 도면 데이터 처리 |

### ComfyUI 심화 — 왜 아티스트에게 중요한가

```
ComfyUI = 이미지 생성의 "포토샵"

일반 AI 이미지 생성: "예쁜 풍경 그려줘" → 나오는 대로 받기
ComfyUI:           모든 단계를 직접 제어

[노드 기반 워크플로우]
텍스트 입력 → 모델 선택 → 샘플러 설정 → 컨트롤넷 → 후처리 → 출력
     ↑            ↑            ↑            ↑
  내가 제어     내가 선택     내가 조절     내가 추가
```

#### ComfyUI에서 제어 가능한 것들

| 제어 항목 | 설명 | 아티스트에게 의미 |
|----------|------|----------------|
| **모델 선택** | SD, SDXL, Flux 등 기본 모델 교체 | 결과물 스타일 근본부터 변경 |
| **LoRA 적용** | 특정 스타일 추가 학습 모델 | 내 스타일로 생성 가능 |
| **컨트롤넷** | 포즈, 깊이, 윤곽선 참조 | 구도/포즈를 정확히 지정 |
| **샘플러 설정** | 생성 과정 세부 조절 | 디테일/분위기 미세 조정 |
| **후처리** | 업스케일, 인페인팅, 부분 수정 | 특정 부분만 수정 가능 |

### LoRA (Low-Rank Adaptation) — 세부 모델

```
기본 모델 = 범용 (모든 걸 평균적으로 잘함)
LoRA     = 특정 스타일/주제에 특화된 "추가 학습"

구조:
  기본 모델 (SD/Flux) + LoRA = 특화된 결과물
  
  하나의 기본 모델에 여러 LoRA를 바꿔 끼울 수 있음
  = 렌즈 교환식 카메라와 같은 개념
```

#### 아티스트가 쓸 수 있는 LoRA 종류

| LoRA 유형 | 효과 | 예시 |
|----------|------|------|
| **애니메이션 LoRA** | 지브리/셀 애니메이션 스타일 | 무드보드에 애니 터치 |
| **건축 LoRA** | 건축 렌더링 특화 | 공간 기획 시각화 |
| **인물 LoRA** | 특정 인물 스타일 | 포트레이트 레퍼런스 |
| **조명 LoRA** | 특정 조명 분위기 | 촬영 라이팅 시뮬레이션 |
| **필름 LoRA** | 35mm 필름 질감 | 빈티지 무드보드 |

```
아티스트에게 가장 중요한 점:
→ "내 스타일"을 학습시킬 수 있다
→ 내 작업물로 LoRA를 만들면 → AI가 내 스타일로 생성
→ = 나만의 AI 도구가 되는 것
```

#### LoRA를 찾는 곳

| 사이트 | 주소 | 특징 |
|-------|------|------|
| **Civitai** | civitai.com | 최대 규모, 리뷰/예시 풍부 |
| **Hugging Face** | huggingface.co | 기술적, 최신 모델 빠름 |

### Stable Diffusion vs Flux

ComfyUI에서 돌리는 **기본 모델**이 두 종류다.

| | **Stable Diffusion (SD)** | **Flux** |
|---|---|---|
| **역사** | 가장 오래됨 | 최신 |
| **버전** | SD 1.5 → SDXL → SD3 | Flux.1 Dev / Schnell |
| **커뮤니티** | 가장 큼 | 성장 중 |
| **LoRA 생태계** | 풍부 (수만 개) | 성장 중 (수천 개) |
| **텍스트 렌더링** | 약함 | **우수** |
| **이미지 퀄리티** | 좋음 | **더 좋음** |
| **선택 기준** | 다양한 LoRA가 필요할 때 | 최신 퀄리티가 필요할 때 |

```
결론:
  처음 시작 → SD (자료/튜토리얼 많음)
  퀄리티 우선 → Flux (최신, 깨끗)
  둘 다 ComfyUI에서 돌릴 수 있음
```

## 1-6. MCP

### MCP = AI를 위한 "만능 USB-C 포트"

```
MCP (Model Context Protocol)
= AI가 외부 도구에 직접 접근하는 표준 규격

MCP 없이:
  나 → 구글 드라이브에서 파일 다운로드 
    → 복사 → AI에 붙여넣기 
    → 답변 복사 → 노션에 붙여넣기
  = 매번 수동으로 옮기기

MCP 있으면:
  나 → "구글 드라이브에서 그 파일 읽고, 정리해서 노션에 저장해줘"
  AI → (직접 접근 → 처리 → 저장)
  = AI가 내 도구에 직접 손을 대는 것
```

---

### 아티스트가 연결할 수 있는 것들

| 연결 대상 | 뭘 할 수 있나 | MCP 서버 이름 |
|----------|-------------|-------------|
| **구글 드라이브** | 파일 검색, 읽기, 저장 | Google Drive MCP |
| **노션** | 페이지 조회/수정, DB 업데이트 | Notion MCP |
| **피그마** | 디자인 분석, 코드 변환 | Figma MCP |
| **블렌더** | 3D 모델 제어, 최적화 | Blender MCP |
| **파일 시스템** | 폴더 정리, 파일 이름 변경 | Filesystem MCP |
| **웹 브라우저** | 웹 검색, 스크래핑 | Puppeteer MCP |
| **캘린더** | 일정 조회/추가 | Google Calendar MCP |

---

### MCP 설치 방법

```
겁먹지 않아도 된다. 4단계면 끝.

1. Cursor/Antigravity 또는 Claude Desktop 설치 (이미 했으면 건너뛰기)
2. 설정에서 MCP 서버 추가
3. 연결할 서비스 선택 (예: 구글 드라이브)
4. 인증 (로그인)
5. 끝. 이제 AI에게 "드라이브에서 ~ 찾아줘" 가능
```

#### 도구별 MCP 설정 위치

| 도구 | MCP 설정 위치 |
|------|-------------|
| **Cursor** | 설정 → MCP Servers |
| **Antigravity** | 설정 → MCP Servers |
| **Claude Desktop** | 설정 파일 (`claude_desktop_config.json`) |
| **Claude Code** | 설정 또는 프로젝트 폴더 내 설정 |

---

### MCP 서버 찾는 곳

| 사이트 | 주소 | 설명 |
|-------|------|------|
| **cursor.directory** | cursor.directory → MCP Servers 탭 | 인기 서버 큐레이션 |
| **mcp.so** | mcp.so | MCP 서버 검색 엔진 |
| **GitHub** | github.com → "awesome-mcp-servers" 검색 | 전체 목록 |
| **Anthropic 공식** | modelcontextprotocol.io | 공식 문서 + 스펙 |

### 아티스트 실전 MCP 예시

#### 사진작가

```
연결: 구글 드라이브 MCP + 노션 MCP

명령:
  "내 구글 드라이브 촬영 폴더에서 지난달 파일들 →
   날짜별로 정리해서 노션 DB에 넣어줘"

결과:
  AI가 드라이브 접근 → 파일 목록 수집 → 날짜 파싱 → 노션 DB에 행 추가
  = 수동으로 하면 1시간, MCP로 하면 30초
```

#### 영상제작자

```
연결: 피그마 MCP + 파일 시스템 MCP

명령:
  "이 피그마 디자인 → 웹 포트폴리오 코드로 변환해줘"

결과:
  AI가 피그마 디자인 분석 → HTML/CSS 코드 생성 → 로컬 파일로 저장
```

#### 공간기획자

```
연결: 노션 MCP

명령:
  "노션에 있는 프로젝트 DB에서
   예산 3000만원 이상 프로젝트만 뽑아서 정리해줘"

결과:
  AI가 노션 DB 조회 → 필터링 → 정리된 표 출력
```

#### 댄서/안무가

```
연결: 캘린더 MCP + 파일 시스템 MCP

명령:
  "이번 달 연습 일정을 캘린더에서 가져와서
   연습 기록 폴더와 매칭해줘"

결과:
  AI가 캘린더 조회 → 파일 폴더 스캔 → 날짜별 매칭표 생성
```

### MCP 핵심 제약

> **MCP는 설치형(Cursor / Antigravity / Claude Desktop / Claude Code) 전용.**
> 웹 버전(claude.ai, chatgpt.com)에서는 사용 불가.
> = 설치형을 써야 하는 가장 큰 이유 중 하나.
