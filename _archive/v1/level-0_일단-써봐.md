## 1. 모델을 바꿔라 (가장 중요)

### 문제: 대부분의 사람이 무료/낮은 모델을 쓰고 있다

AI 서비스에 접속하면 기본으로 설정된 모델이 있습니다.
이 기본 모델은 대부분 **가장 낮은 등급**입니다.
빠르지만, 깊이가 없습니다.

여러분이 "AI가 별로다"라고 느끼는 이유의 **절반은 여기에 있습니다.**

### 실제 사례

어떤 대표님이 Gemini를 매일 쓰고 있었습니다.
사업계획서를 검토하고, 아이디어를 정리하고, 문서도 만들었습니다.
그런데 결과가 항상 **두루뭉술**했습니다.

확인해보니, 모델이 **"중간"**으로 설정되어 있었습니다.

같은 질문을 **"높은"** 모델로 바꿔서 다시 물어봤습니다.
결과가 완전히 달랐습니다.

> **"높은 애를 쓰세요. 그럼 돼요."**

정말 이게 다였습니다.

---

### 각 서비스별 모델 선택 방법

| 서비스 | 어디서 바꾸나 | 추천 모델 | 비고 |
|--------|------------|----------|------|
| **ChatGPT** | 대화창 상단 드롭다운 | GPT-4o | 무료 사용자도 제한적 사용 가능 |
| **Gemini** | 대화창 상단 모델 선택 | Gemini 2.5 Pro | 무료는 Flash만 가능 |
| **Claude** | 대화창 상단 모델 선택 | Claude Sonnet 4 | 무료 제한 있음 |
| **Perplexity** | 설정에서 모델 변경 | Pro Search | 무료는 기본 검색만 |

> [!TIP]
> **모델을 바꾸는 데 10초면 됩니다.** 지금 즉시 해보세요.
> 대화창 상단에 모델 이름이 보이면, 그걸 누르고 가장 높은 것을 선택하면 됩니다.

---

### 모델 등급 비유

AI 모델을 직원에 비유하면 이렇습니다:

```
낮은 모델 = 인턴에게 시키는 것   → 빠르지만 얕다
중간 모델 = 대리에게 시키는 것   → 무난하다
높은 모델 = 팀장에게 시키는 것   → 느리지만 깊다
```

여러분이 중요한 사업계획서를 쓸 때, 인턴에게 시키시겠습니까?
**아닙니다. 팀장에게 시킵니다.**

AI도 똑같습니다.
중요한 일에는 **높은 모델을 쓰세요.**

---

### Before / After 예시

**낮은 모델**에게 "사업계획서 요약해줘"라고 했을 때:

> "이 사업계획서는 혁신적인 사업 모델을 제시하며, 지속 가능한 성장을 목표로 합니다. 주요 사업 영역은 해양수산 분야이며, 향후 시장 확대가 기대됩니다."

…읽어도 뭔지 모르겠습니다. **뻔한 요약**입니다.

---

**높은 모델**에게 같은 요청을 했을 때:

> **핵심 리스크 3가지:**
> 1. **시장 규모 검증 미흡** — TAM/SAM/SOM 구분 없이 전체 시장 규모만 제시. 실제 도달 가능한 시장이 불분명
> 2. **수익 모델의 단가 근거 부재** — 월 구독료 ₩50,000의 산출 근거가 없음. 경쟁사 가격 대비 30% 높은데 이유 미설명
> 3. **경쟁사 대비 차별점 불명확** — "AI 기반"이라고만 되어 있고, 기존 솔루션과의 기술적 차이가 구체적이지 않음

같은 질문입니다. **모델만 바꿨습니다.**
결과가 이렇게 다릅니다.

---

### 결제 가이드: 뭘 결제해야 하는가

> [!IMPORTANT]
> **무료 모델의 한계를 정확히 아셔야 합니다.**
> 무료로 쓰면 낮은 모델만 쓸 수 있거나, 높은 모델의 사용 횟수가 극도로 제한됩니다.

## 2. 말을 제대로 해라

### 문제: AI에게 말을 안 하고 화를 낸다

AI가 이상한 답을 내면 대부분 이렇게 반응합니다:

*"아 왜 이상한 답을 주는 거야!"*

그런데 돌이켜보면, **내가 이상하게 물어본 겁니다.**

### 실제 사례

한 대표님이 회의 녹음을 AI에 넣고 데이터를 뽑고 있었습니다.
그런데 결과에 "8,000평"이라는 숫자가 빠져 있었습니다.

대표님은 화를 냈습니다.
*"나는 분명히 8,000평이라고 말했는데!"*

실제 녹취를 다시 확인했습니다.
**그런 말이 없었습니다.**

> **"틀린 말을 하지 않아 놓고 화를 내시면 안 돼요.**
> **인간한테도 마찬가지고, AI한테도 마찬가지."**

AI는 마법사가 아닙니다.
**여러분이 말한 것만 알 수 있습니다.**
말하지 않은 것을 맞추길 기대하면 안 됩니다.

---

### 좋은 입력 vs 나쁜 입력 (5개 비교)

| ❌ 나쁜 입력 | ✅ 좋은 입력 |
|-------------|-------------|
| "사업계획서 써줘" | "해양수산 분야 창업지원 프로그램 사업계획서. 심사위원은 공무원 3명, 업계 전문가 2명. 배점 기준은 [첨부]. 분량 A4 10장." |
| "이거 고쳐줘" | "이 표에서 윤빌리지 → 윤플레이스로 이름을 바꾸고, 면적 800평 → 8,000평으로 수정해줘." |
| "함수가 안 돼" | "엑셀에서 VLOOKUP 함수를 썼는데 #N/A 에러가 나요. 범위는 A1:D100이고, 찾는 값은 '해양'입니다." |
| "좋은 이름 추천해줘" | "친환경 수산물 브랜드 이름. 타겟: 30대 주부. 키워드: 신선, 로컬, 투명한 유통. 후보 5개." |
| "번역해줘" | "이 계약서를 한→영 번역. 법률 용어는 원문 병기. 의역보다 직역 우선." |

패턴이 보이시나요?

**나쁜 입력**은 한 줄입니다. 맥락이 없습니다.
**좋은 입력**은 상황, 조건, 형식이 들어가 있습니다.

---

### 핵심 원칙 3가지

AI에게 요청할 때 이 세 가지만 기억하세요:

#### ① 역할을 줘라

```
"너는 해양수산부 창업지원 심사위원이다."
"너는 10년 경력 카피라이터다."
"너는 세무사다."
```

역할을 주면 AI가 **그 관점에서** 답합니다.
"사업계획서 봐줘"와 "너는 심사위원인데, 이 사업계획서를 평가해줘"는 결과가 완전히 다릅니다.

#### ② 맥락을 줘라

```
"상황: 우리 회사는 직원 5명의 스타트업이고, 첫 정부과제에 도전한다."
"배경: 이 보고서는 이사회에 제출할 것이다."
"독자: IT를 모르는 50대 경영진이다."
```

같은 보고서도 **누가 읽느냐**에 따라 내용이 달라져야 합니다.
AI에게 그걸 알려주세요.

#### ③ 형식을 줘라

```
"표 형태로 정리해줘"
"불릿 포인트 5개로"
"A4 1장 분량으로"
"이메일 형식으로"
```

형식을 지정하지 않으면 AI는 장문으로 답합니다.
**원하는 형태를 말해주세요.**

---

### 이 세 가지를 합치면 이렇게 됩니다

> ❌ **"사업계획서 써줘"**

> ✅ **"너는 해양수산부 창업지원 심사위원이다. 아래 사업계획서를 읽고, 심사 기준(혁신성 30점, 실현가능성 40점, 팀역량 30점)에 따라 각 항목별 점수와 개선점을 표로 정리해줘. 실현가능성 점수가 낮을 것 같으면 어떻게 보완할 수 있는지도 알려줘."**

두 요청 중 어떤 게 더 좋은 답을 받을까요?
당연히 후자입니다.

## 3. 녹음부터 시작해라 (가장 쉬운 시작)

### 왜 녹음인가

AI를 쓰려면 뭔가를 **입력**해야 합니다.
그런데 타이핑이 싫은 분들이 많습니다.

**말은 할 수 있잖아요.**

회의를 하고, 통화를 하고, 브레인스토밍을 합니다.
이걸 **녹음해서 AI에 넣으면**, 타이핑 없이도 AI를 쓸 수 있습니다.

### 실제 사례: NotebookLM으로 회의 녹음 정리하기

> **"녹음 파일은 여기 올려서 하는 게 제일 싸게 쳐요."**

한 대표님이 40분짜리 회의를 녹음했습니다.
이 파일을 **Google NotebookLM**에 올렸습니다.

결과:
- 40분 녹음 → **텍스트 자동 추출**
- 텍스트에서 → **마인드맵 자동 생성**
- 핵심 내용 → **요약 자동 완성**

전체 과정: **약 5분.**
직접 회의록 쓰면 **1시간 넘게** 걸릴 일이었습니다.

---

### 단계별 방법

> [!NOTE]
> NotebookLM은 **완전 무료**입니다.
> Google 계정만 있으면 바로 사용 가능합니다.

**Step 1.** 핸드폰으로 녹음합니다
- iPhone: 기본 '음성 메모' 앱
- Android: 기본 '녹음기' 앱
- 별도 앱 필요 없습니다

**Step 2.** Google NotebookLM에 접속합니다
- 주소: **notebooklm.google.com**
- Google 계정으로 로그인

**Step 3.** 새 노트북 만들기
- "새 노트북" 클릭
- "오디오 파일 업로드" 선택

**Step 4.** 녹음 파일 업로드
- 녹음한 파일을 끌어다 놓거나 선택해서 업로드
- 잠시 기다리면 텍스트로 변환됨

**Step 5.** 활용하기
- "요약해줘" → 전체 내용 요약
- "핵심 결정사항만 뽑아줘" → 액션 아이템 추출
- "마인드맵으로 정리해줘" → 구조화된 시각 자료

**Step 6.** 틀린 부분만 수정
- AI가 100% 정확하지는 않습니다
- 이름, 숫자, 고유명사를 확인하고 수정하세요

---

### 마인드맵 예시

실제로 회의 녹음을 NotebookLM에 올리면 이런 식으로 정리됩니다:

```
회의 주제: 윤팜 부지 개발
├── 현황
│   ├── 부지: 8,000평 (임야)
│   ├── 보유 토지: 120평
│   └── 항공사진 보유
├── 개발 방향
│   ├── 윤플레이스 (200평)
│   └── 건축비 예정
└── 이슈
    ├── 맹지 여부 확인 필요
    └── 농지 전용 절차
```

40분 동안 여기저기 튀었던 회의 내용이
**한눈에 보이는 구조**로 정리됩니다.

이걸 손으로 하면 1시간입니다.
AI에게 시키면 **5분**입니다.

## 4. 체크리스트: 오늘 당장 할 것

이 문서를 읽었다면, 지금 바로 아래 네 가지를 해보세요.
**10분이면 됩니다.**

```
□ 내가 쓰는 AI 서비스에서 모델을 "높은 것"으로 바꿨다
□ 하나의 서비스라도 유료 결제했다 (또는 결제할 서비스를 정했다)
□ 녹음 파일 하나를 NotebookLM에 올려봤다
□ AI에게 요청할 때 "역할 + 맥락 + 형식"을 넣어봤다
```

네 개 중 **하나라도** 했으면 성공입니다.
오늘 하나, 내일 하나. 그렇게 시작하면 됩니다.

| Gemini Advanced | ❌ 학습 안 함 (명시) | ⚠️ 서비스 개선에 사용될 수 있음 |

유료 플랜을 쓰면 이 부분은 크게 걱정하지 않아도 됩니다.
무료 플랜은 서비스마다 다르므로, 설정을 확인하세요.

### Q: 어떤 AI가 제일 좋은가요?

**A:** 용도에 따라 다릅니다. **"제일 좋은 AI"는 없습니다.** "내 용도에 맞는 AI"가 있을 뿐입니다.

| 용도 | 추천 AI | 이유 |
|------|---------|------|
| 이미지 만들기 | **ChatGPT** | 이미지 생성이 가장 자연스러움 |
| 녹음 정리 / 마인드맵 | **Gemini (NotebookLM)** | 무료, 음성 처리 우수 |
| 정밀한 문서 작업 | **Claude** | 구조화 능력, 디테일 최고 |
| 검색 + 답변 | **Perplexity** | 출처를 같이 보여줌 |
| 일반적인 대화 / 질문 | **ChatGPT** | 가장 범용적 |

> [!TIP]
> 헷갈리면 **ChatGPT Plus**부터 시작하세요. 가장 범용적입니다.
> 익숙해지면 용도별로 나눠 쓰면 됩니다.

### Q: 보안이 걱정됩니다

**A:** 당연히 조심해야 합니다. **하지만 과하게 두려워할 필요는 없습니다.**

#### ❌ 절대 넣지 마세요
- 주민등록번호
- 계좌번호, 카드번호
- 비밀번호
- 개인 의료 정보
- 고객의 개인정보

#### ✅ 넣어도 괜찮습니다
- 사업 아이디어 (아이디어는 실행이 핵심이지, 비밀이 아닙니다)
- 공개된 사업 정보
- 일반적인 업무 문서 초안
- 회의 내용 (민감 정보 제외)

> [!WARNING]
> **원칙: "이 내용이 유출되면 큰일 나는가?"를 기준으로 판단하세요.**
> 큰일 나는 정보 → 넣지 마세요.
> 아이디어 수준 → 괜찮습니다.

### Q: AI가 거짓말을 하면 어떡하나요?

**A:** AI는 거짓말을 합니다. 이걸 **"환각(hallucination)"**이라고 부릅니다.

특히 주의해야 할 상황:
- **숫자**: 통계, 시장 규모, 법률 조항 번호 → 반드시 확인
- **사람 이름**: 존재하지 않는 사람을 만들어내기도 함
- **최신 정보**: 학습 데이터 이후의 정보는 모를 수 있음

대처법:
1. 중요한 숫자는 **Perplexity**로 팩트체크
2. "출처를 알려줘"라고 항상 요청
3. AI의 답을 **초안**으로만 취급하고, 최종 확인은 사람이
