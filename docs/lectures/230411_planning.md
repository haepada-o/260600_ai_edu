# 기획서 작성: AI를 활용한 정부지원 및 비즈니스 기획안 빌딩
> **대상**: 예비 창업자, 스타트업 기획자, 문화 예술 기획자 | **시간**: 2시간 특강 | **핵심**: ChatGPT를 비롯한 LLM을 비서가 아닌 '평가 위원' 및 '기획 파트너'로 빙의시켜 사업 계획서, 예산 계획, 마케팅 로드맵 초안을 단시간에 빌딩하는 프롬프트 테크닉.

---

## 📌 비즈니스 기획안 작성 로드맵 (4단계)

이 가이드는 실제 강연에서 소개된 기획 문서 고속 생성 프롬프트 시나리오를 바탕으로 작성된 기획자 맞춤형 실전 매뉴얼입니다.

---

### Step 1. 기획 파트너로서의 역할(Role) 및 맥락(Context) 설정 (20분)
- **판단 (Trigger):** AI에게 단순하게 "지원사업 계획서 양식 짜줘"라고 물어서 초등학생 수준의 단순한 목차만 리턴받았을 때.
- **할 일 (Actions):**
  1. 기획서 작성을 시작하기 전에 AI에게 **평가 위원의 전문적 시선(Role)**을 부여한다.
  2. 기획안이 제출될 기관의 성격(예: 창업진흥원, 한국콘텐츠진흥원)과 사업의 예산 규모, 목표를 사전에 주입한다.
  3. 맥락 설계 **상·생·방·제·소** 5요소로 요청을 구성한다.
- **실전 페르소나 설정 프롬프트:**
  ```
  너는 스타트업 정부지원사업(예: 예비창업패키지)의 수석 심사위원이야.
  기술적 실현 가능성, 사업성, 예산 집행 계획의 타당성을 극도로 꼼꼼하게 평가하는 성향을 가졌어.
  
  내가 지금 제안하려는 아이디어에 대해 심사위원의 관점에서 먼저 3가지 질문을 던져줘. 질문에 답하며 기획을 빌딩해 나갈 거야.
  ```

- **[심화] 상·생·방·제·소 5요소 적용 버전:**
  ```
  [상황] 나는 해양수산 분야 창업 지원사업에 제출할 8개월 과제 기획서를 써야 한다.
  [생각] 우리 팀의 가장 큰 강점은 실제 투자 유치 및 집행 실적이 있다는 것이다.
  [방향] 심사위원이 실현 가능성과 팀 수행 역량을 확신할 수 있는 구조로 구성하고 싶다.
  [제약] 과장된 수사어 금지. 수치는 내가 제공한 것만 쓸 것.
  [소스] 참고: 창업진흥원 2025년도 예비창업패키지 공고문 (첨부)
  ```

---

### Step 2. 아이디어 5대 핵심 요소로 해체하기 + Prompt Chaining (30분)
- **판단 (Trigger):** 머릿속에 사업 아이디어는 있으나, 이를 비즈니스 모델 규격(Problem-Solution-Customer)에 맞게 정제하기 어려울 때.
- **할 일 (Actions):**
  1. 기획서를 **한 번에 써달라 하지 말고**, Prompt Chaining으로 단계별 분해한다.
  2. **[체인 1]** 아이디어 구조화 → **[체인 2]** 논리 검증 → **[체인 3]** 본문 작성 순으로 진행한다.

- **Prompt Chaining 기획서 체인:**
  ```
  # 체인 1: 아이디어 구조화
  내 사업 아이디어는 [내 아이디어 한 줄 요약]이다.
  이 아이디어를 다음 5가지 항목으로 구조화해줘:
  1. 고객의 문제 정의 (Pain Point)
  2. 해결 방안 (Solution)
  3. 핵심 타겟 고객층 (Target Customer)
  4. 수익 모델 (Revenue Stream)
  5. 초기 모객 채널 (Marketing Channel)
  ```
  ```
  # 체인 2: CoT 논리 검증 (체인 1 결과물 붙여넣기 후)
  위 구조를 검토할 때, ①Pain Point가 실제 데이터로 뒷받침되는지, 
  ②Solution이 Pain Point를 직접 해소하는지, ③수익 모델이 현실적인지 
  단계별로 순서대로 분석해줘.
  ```
  ```
  # 체인 3: 본문 집필 (검증 완료 후)
  검증된 구조를 바탕으로 '사업 타당성' 섹션을 
  공모 기획서 격식체 톤으로 A4 1장 분량으로 완성해줘.
  ```

- **처음이라면 (Tip):** 기획서 전체를 한 번에 쓰게 하면 품질이 들쑥날쑥합니다. 반드시 체인 1 → 2 → 3 순서로 각 단계 결과를 확인하며 진행하세요.

---

### Step 3. 공공 예산 구조 및 마일스톤 설계 자동화 (40분)
- **판단 (Trigger):** 정부 예산이나 지자체 보조금 규격에 맞는 비목(인건비, 외주가공비, 재료비, 광고선전비) 분류 및 현실성 있는 마일스톤(일정)을 작성해야 할 때.
- **할 일 (Actions):**
  1. 전체 사업 기간(예: 8개월)과 총 예산 규모(예: 5천만 원)를 입력한다.
  2. 공공 지원사업 기준에 맞는 비율로 예산 배정 시뮬레이션을 요청한다.
- **예산 기획 프롬프트 (XML 구조 버전 — Claude 권장):**
  ```xml
  <context>
  나는 소프트웨어 개발 아웃소싱이 포함된 스타트업 대표이며,
  정부 공모 지원사업 기획서의 예산 편성을 준비 중이다.
  공공 기관 예산 편성 가이드라인을 따라야 한다.
  </context>

  <instruction>
  총 사업 기간 8개월, 총 정부지원금 5,000만 원 기준으로
  비목별 예산 배분표를 작성해줘.
  </instruction>

  <data>
  - 비목: 인건비, 외주가공비, 마케팅비, 운영비
  - 인건비 비중 제한: 총액의 50% 미만
  - 일반 관리비 제한: 10% 미만
  </data>

  <output_format>
  마크다운 표 형식 (비목 / 금액 / 비율 / 책정 사유).
  마지막 행에 합계 행 포함.
  </output_format>
  ```

---

### Step 4. 자가 진단 및 악플러/독설 피드백 요청 (30분)
- **판단 (Trigger):** 작성된 초안의 논리적 맹점을 사전에 파악하여 실제 대면 심사나 서류 심사에서 탈락하는 리스크를 낮추고 싶을 때.
- **할 일 (Actions):**
  1. 완성된 기획서 텍스트를 AI에게 통째로 입력한다.
  2. AI에게 **'가장 까다롭고 비판적인 심사역'** 또는 **'악플러 독자'** 역할을 강제하여 계획서의 맹점 3가지를 공격하라고 주문한다.
  3. **CoT(단계적 사고)** 를 추가하면 더 날카로운 지적이 나온다.
- **실전 검증 독설 프롬프트 (CoT 적용):**
  ```
  [사업계획서 초안]: [작성된 사업계획서 복사 붙여넣기]
  
  너는 경쟁사의 수석 기획자이자, 나를 탈락시켜야 하는 심사위원이야.
  위 계획서를 검토할 때, ①논리적 모순 → ②시장 진입 장벽 실현 가능성 →
  ③예산 낭비 요소 순으로 단계별로 분석한 뒤, 각각 가장 심각한 문제 하나씩을 지적해줘.
  칭찬은 필요 없어. 단점만 공격해.
  ```

- **Few-shot 피드백 강도 조절:**
  ```
  아래 예시처럼 날카롭게 비판해줘.
  
  예시) "3페이지 수익 모델의 '구독료 월 5만원' 설정은 
  타겟 고객(자영업자)의 평균 소프트웨어 지출 예산(월 1~2만원)과 
  3~5배 괴리가 있어 현실성이 없음."
  
  위 수준으로 내 계획서의 가장 취약한 부분 3곳을 지적해줘.
  ```
